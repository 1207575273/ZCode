# @ 文件引用系统实现总结

> 日期: 2026-03-14

## 一、功能概述

为 ZCli 增加 `@` 文件引用能力，用户在输入框中输入 `@` 即可浏览项目文件、模糊搜索、引用文件路径，LLM 侧自动感知被引用文件的位置。

**交互流程：**
1. 用户输入 `@` → 展示当前目录结构（浏览模式）
2. 输入 `@src/` → 展开 `src/` 目录内容
3. 输入 `@main` → 模糊搜索匹配 `main` 的文件
4. Enter/Tab 选中目录 → 导航进入；选中文件 → 补全路径
5. 提交时自动解析 `@path`，注入 `<file-references>` XML 告知 LLM 文件位置

## 二、架构设计

### 模块划分

```
src/file-index/          # 项目文件树维护与查询（高内聚独立模块）
├── file-index.ts        # FileIndex — 全量扫描 + 增量更新 + 模糊搜索 + 目录浏览
├── file-watcher.ts      # FileWatcher — fs.watch recursive 文件监听
├── ignore-rules.ts      # 合并 builtin + .gitignore + .zcliignore 忽略规则
├── types.ts             # SearchResult / DirEntry 类型定义
└── index.ts             # 统一导出

src/utils/
└── at-resolver.ts       # AtResolver — 用户输入 @path 解析 + LLM context 生成

src/ui/
└── AtSuggestion.tsx     # @ 建议浮层（纯显示组件）
```

### 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 文件监听方案 | `fs.watch` recursive | 零原生依赖，Node/Bun 通用，无需编译 |
| 模糊搜索引擎 | `fzf-for-js` | 与 fzf 排序算法一致，结果质量高 |
| LLM 注入策略 | 仅告知位置，LLM 自行 Read | 不发送文件内容，节省 token，LLM 自由决策 |
| 目录浏览实现 | 基于已有索引单次遍历 | 无额外磁盘 IO，O(n) 复杂度 |
| AtResolver 归属 | `src/utils/` | 纯解析工具，不含业务逻辑，不污染 `core/` |

### 模块职责边界

- **`file-index/`** — 只关心「项目有哪些文件」：扫描、监听、索引、查询
- **`AtResolver`** — 只关心「把 @path 翻译成 LLM 上下文」：正则解析 + XML 拼接
- **`AtSuggestion`** — 只关心「渲染建议列表」：纯展示，无状态，无 useInput
- **`App.tsx`** — 编排层：检测 @ 触发 → 决定浏览/搜索模式 → 管理键盘导航 → 提交时调用 AtResolver

## 三、核心实现细节

### 3.1 FileIndex 数据结构

- 维护排序后的 `string[]`（相对路径，正斜杠分隔）
- 增量操作（add/remove）使用二分查找保持有序，O(log n) 定位
- 每次变更后重建 fzf 实例（fzf 不支持增量更新，但重建成本低）

### 3.2 目录浏览（listEntries）

单次遍历 `paths` 数组，按前缀过滤，拆分首段判断是文件还是目录：

- 有 `/` → 子目录，加入 `Set<string>` 去重
- 无 `/` → 直接文件
- 返回结果：目录在前，文件在后，各自排序

### 3.3 两种 @ 建议模式

| 条件 | 模式 | 数据源 |
|------|------|--------|
| query 为空或以 `/` 结尾 | 浏览模式 | `fileIndex.listEntries(dirPrefix)` |
| query 有文本 | 搜索模式 | `fileIndex.search(query)` |

### 3.4 键盘交互

- `↑↓` — 循环导航建议列表
- `Tab` / `Enter` — 目录：导航进入（`@src/`）；文件：补全路径 + 空格
- `Escape` — 清空输入

### 3.5 AtResolver 解析规则

正则 `(?:^|\s)@([\w./\\-]+(?:#\d+(?:-\d+))?)` 匹配：
- `@src/main.ts` — 普通文件引用
- `@src/main.ts#10` — 指定行号
- `@src/main.ts#10-20` — 指定行范围

生成 XML：
```xml
<file-references>
<file path="src/main.ts" absolute="D:\project\src\main.ts" />
<file path="src/utils.ts" absolute="D:\project\src\utils.ts" lines="10-20" />
</file-references>
```

## 五、附带修复

- **CommandSuggestion.tsx** — 修复 `exactOptionalPropertyTypes` 下 `color` 属性不接受 `undefined` 的 TS 编译错误
- **terminal-screen.ts** — 清屏优化（`\x1b[2J\x1b[3J\x1b[H`），解决 WelcomeScreen 残留和 resize 伪影
- **useTerminalSize.ts** — 终端 resize 防抖 + 彻底清屏

## 六、遗留事项

- `file-index` 模块尚无单元测试覆盖，建议补充 `listEntries`、`search`、`add/remove` 的用例
- `AtResolver` 可补充边界用例测试（多个 @、行范围、路径含特殊字符等）
