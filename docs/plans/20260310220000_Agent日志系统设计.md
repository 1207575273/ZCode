# F9 Agent 日志系统 — 设计文档

> 日期: 2026-03-10 | 优先级: P1 | 前置依赖: F8 会话持久化

---

## 1. 目标

在不侵入业务代码的前提下，记录 Agent 每轮执行的完整过程：LLM 调用、工具执行、MCP 连接、权限授予、异常、会话结束汇总。支持日志审计和 WebUI 对话还原。

---

## 2. 架构：事件流 + 观察者

```
AgentLoop (AsyncGenerator<AgentEvent>)
    │
    ├──→ useChat (UI 消费者：更新界面)        ← 已有
    │
    └──→ SessionLogger (日志消费者：写入 JSONL)  ← 新增
```

**核心原则：**
- AgentLoop / Provider / McpManager 只负责 **emit 事件**，不感知日志存在
- `SessionLogger` 作为独立观察者订阅事件流，负责格式化和持久化
- 业务代码零侵入，日志逻辑集中在一处
- 测试时不挂 Observer 即可，零 mock

---

## 3. 存储

复用 F8 的 session JSONL 文件，在现有 `~/.ccode/sessions/<project-slug>/` 目录中追加更多事件类型。

**理由：**
- session JSONL 已有 append 基础设施和事件树结构
- `loadMessages()` 已自动跳过非 user/assistant 事件，零影响
- 一个会话一个文件，无跨文件关联复杂度
- `parentUuid` 事件链天然支持事件排序和关联

## 4. 事件类型清单

### 4.1 已有事件（不变）

| 事件 | 说明 |
|------|------|
| `session_start` | 会话创建 |
| `session_resume` | 会话恢复 |
| `user` | 用户消息 |
| `assistant` | LLM 回复 |
| `system` | 系统消息 |

### 4.2 新增观测事件

| 事件 | 时机 | 关键字段 |
|------|------|---------|
| `llm_call_start` | LLM 调用前 | provider, model, messageCount |
| `llm_call_end` | LLM 调用后 | inputTokens, outputTokens, durationMs, stopReason, error? |
| `tool_call_start` | 工具执行前 | toolName, toolCallId, args |
| `tool_call_end` | 工具执行后 | toolCallId, durationMs, success, resultSummary, error? |
| `mcp_connect_start` | MCP 连接前 | serverName, transport |
| `mcp_connect_end` | MCP 连接后 | serverName, success, toolCount, durationMs, error? |
| `tool_fallback` | 工具降级 | toolName, fromLevel, toLevel, reason |
| `permission_grant` | 权限授予 | toolName, always |
| `error` | 异常 | source, message, stack |
| `session_end` | 会话结束 | totalInputTokens, totalOutputTokens, totalToolCalls, totalDurationMs |

### 4.3 异常标记规则

所有 `*_end` 事件携带可选 `error` 字段：
- 正常完成：`error` 为 `undefined`
- 异常终止：`error` 包含错误信息，`success: false`
- 部分完成（如流式中断）：`error` 包含原因，已接收的 token 数仍记录

```typescript
// llm_call_end 示例 — 正常
{ type: 'llm_call_end', durationMs: 2340, inputTokens: 1520, outputTokens: 380, stopReason: 'end_turn' }

// llm_call_end 示例 — 异常
{ type: 'llm_call_end', durationMs: 890, inputTokens: 1520, outputTokens: 42, stopReason: 'error', error: 'Connection reset' }

// llm_call_end 示例 — 用户中止
{ type: 'llm_call_end', durationMs: 1200, inputTokens: 1520, outputTokens: 156, stopReason: 'abort', error: 'User aborted' }
```

## 5. Provider 流式事件改造

### 5.1 现状

```typescript
interface LLMProvider {
  chat(messages: Message[], options?: ChatOptions): AsyncGenerator<string>
}
```

只返回文本 chunk，无 token 统计。

### 5.2 改造后

```typescript
/** 流式事件：文本 chunk 或最终 usage 统计 */
type StreamEvent =
  | { type: 'text'; text: string }
  | { type: 'usage'; inputTokens: number; outputTokens: number; stopReason: string }
  | { type: 'error'; error: string; partialOutputTokens?: number }

interface LLMProvider {
  readonly name: string
  chat(messages: Message[], options?: ChatOptions): AsyncGenerator<StreamEvent>
}
```

**关键点：**
- `text` 事件：与现有 `string` yield 等价，AgentLoop 提取 `.text`
- `usage` 事件：流结束前最后一个事件，包含 token 统计（Anthropic/OpenAI SDK 都在最后一个 chunk 返回 usage）
- `error` 事件：流异常时 emit，携带已产生的部分 token 数（`partialOutputTokens`），保证计量不丢失

### 5.3 AgentLoop 适配

```typescript
// 改造前
for await (const text of provider.chat(messages, options)) {
  yield { type: 'text', text }
}

// 改造后
for await (const event of provider.chat(messages, options)) {
  if (event.type === 'text') {
    yield { type: 'text', text: event.text }
  } else if (event.type === 'usage') {
    yield { type: 'llm_usage', ...event }  // 新 AgentEvent 类型
  } else if (event.type === 'error') {
    yield { type: 'llm_error', error: event.error, partialOutputTokens: event.partialOutputTokens }
  }
}
```

## 7. SessionLogger 设计

### 7.1 职责

订阅 AgentEvent 流，转换为 SessionEvent 并写入 JSONL。

### 7.2 接口

```typescript
class SessionLogger {
  /** 当前会话 ID */
  #sessionId: string | null = null
  /** 事件链的最新 UUID（parentUuid） */
  #lastEventUuid: string | null = null
  /** 当前 turn 的累计统计 */
  #turnStats: TurnStats

  /** 绑定到一个会话 */
  bind(sessionId: string): void

  /** 消费一个 AgentEvent，转换并写入 JSONL */
  consume(event: AgentEvent): void

  /** 记录 MCP 连接事件（不经过 AgentLoop） */
  logMcpConnect(phase: 'start' | 'end', info: McpConnectInfo): void

  /** 写入 session_end 汇总事件 */
  finalize(): void
}
```

### 7.3 事件转换映射

| AgentEvent | → SessionEvent |
|-----------|---------------|
| `llm_start` | `llm_call_start` |
| `llm_usage` | `llm_call_end`（正常） |
| `llm_error` | `llm_call_end`（异常标记） |
| `tool_start` | `tool_call_start` |
| `tool_done` | `tool_call_end` |
| `tool_fallback` | `tool_fallback` |
| `permission_grant` | `permission_grant` |
| `error` | `error` |

### 7.4 集成点（useChat.ts）

```typescript
// 现有的 AgentEvent 消费循环
for await (const event of loop.run(history)) {
  // UI 消费（已有）
  if (event.type === 'text') { ... }
  else if (event.type === 'tool_start') { ... }

  // 日志消费（新增一行）
  sessionLogger.consume(event)
}
```

**只在 useChat 中加一行 `sessionLogger.consume(event)`**，其余模块无改动。

### 7.5 TurnStats 累计统计

```typescript
interface TurnStats {
  totalInputTokens: number
  totalOutputTokens: number
  totalToolCalls: number
  totalDurationMs: number
  llmCalls: number
  errors: number
}
```

每轮 submit 开始时重置，`session_end` 或退出时写入汇总。

## 8. McpManager 事件改造

McpManager 的连接过程不经过 AgentLoop，需要单独处理。

### 8.1 方案

在 `McpManager.connectAll()` 中，连接每个 server 前后 emit 事件。McpManager 不直接写日志，而是通过回调通知外部：

```typescript
interface McpConnectEvent {
  phase: 'start' | 'end'
  serverName: string
  transport: string
  success?: boolean
  toolCount?: number
  durationMs?: number
  error?: string
}

class McpManager {
  onConnect?: (event: McpConnectEvent) => void

  async connectAll(): Promise<void> {
    for (const [name, config] of servers) {
      this.onConnect?.({ phase: 'start', serverName: name, transport: config.transport })
      const startTime = Date.now()
      try {
        await this.connect(name, config)
        this.onConnect?.({ phase: 'end', serverName: name, transport: config.transport,
          success: true, toolCount: ..., durationMs: Date.now() - startTime })
      } catch (err) {
        this.onConnect?.({ phase: 'end', serverName: name, transport: config.transport,
          success: false, error: err.message, durationMs: Date.now() - startTime })
      }
    }
  }
}
```

### 8.2 集成

在 `useChat.ts` 的 `ensureMcpInitialized` 中挂载回调：

```typescript
mcpManager.onConnect = (event) => sessionLogger.logMcpConnect(event.phase, event)
```

## 11. 与现有 F8 持久化的关系

F8 的 `useChat.ts` 中有手动的 `appendSessionEvent` 调用（记录 user/assistant）。F9 的 `SessionLogger` 将**接管**这些职责：

- `SessionLogger.consume()` 统一处理所有事件的 JSONL 写入
- 移除 `useChat.ts` 中散落的 `appendSessionEvent` 调用
- `sessionIdRef` / `lastEventUuidRef` / `ensureSession` 迁移到 `SessionLogger` 内部
- `useChat.ts` 只保留 `sessionLogger.consume(event)` 一行

这样 useChat 从「UI 消费 + 持久化」职责收窄为纯「UI 消费」，持久化完全交给 SessionLogger。

## 12. 数据流全景

```
用户输入
  │
  ▼
useChat.submit()
  │
  ├── sessionLogger.consume({ type: 'user', ... })          ← 记录用户消息
  │
  ▼
AgentLoop.run(history)
  │
  ├── yield { type: 'llm_start', provider, model, ... }     ← LLM 调用前
  │
  ├── Provider.chat() → AsyncGenerator<StreamEvent>
  │   ├── yield { type: 'text', text }                       ← 流式文本
  │   ├── yield { type: 'usage', inputTokens, ... }          ← 正常结束
  │   └── yield { type: 'error', error, ... }                ← 异常
  │
  ├── yield { type: 'llm_usage', ... } 或 { type: 'llm_error', ... }
  │
  ├── yield { type: 'tool_start', toolName, args }           ← 工具调用前
  │   ├── Tool.execute() 执行
  │   └── yield { type: 'tool_done', durationMs, success }   ← 工具调用后
  │
  ├── yield { type: 'tool_fallback', ... }                   ← 降级（如有）
  ├── yield { type: 'permission_grant', ... }                ← 权限授予
  │
  └── yield { type: 'done' }
        │
        ▼
  useChat 消费循环
  ├── UI 更新（setStreamingMessage / setToolEvents / ...）
  └── sessionLogger.consume(event)  ← 每个事件都写入 JSONL
        │
        ▼
  SessionLogger 内部
  ├── AgentEvent → SessionEvent 转换
  ├── sessionStore.append(sessionId, sessionEvent)
  └── turnStats 累计
```
