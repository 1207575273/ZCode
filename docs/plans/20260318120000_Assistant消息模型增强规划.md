## 一、背景

### 1.1 LLM API 实际返回了什么

**一次 LLM 响应不只是"一段文本"**——它是一个包含多种内容块的复合对象。

**Anthropic Claude 的响应**：

```json
{
  "role": "assistant",
  "content": [
    { "type": "thinking", "thinking": "让我分析一下这个问题..." },
    { "type": "text", "text": "根据分析，答案是..." },
    { "type": "tool_use", "id": "toolu_xxx", "name": "read_file", "input": {"path": "foo.ts"} }
  ],
  "stop_reason": "tool_use",
  "usage": { "input_tokens": 5974, "output_tokens": 580,
             "cache_read_input_tokens": 0, "cache_creation_input_tokens": 0 }
}
```

**OpenAI/兼容模型的响应**：

```json
{
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "根据分析，答案是...",
      "tool_calls": [{ "id": "call_xxx", "function": {"name": "read_file", "arguments": "..."} }],
      "refusal": null
    },
    "finish_reason": "tool_calls"
  }],
  "usage": { "prompt_tokens": 5974, "completion_tokens": 580 }
}
```

**核心事实**：`assistant 消息体` + `tool_calls` + `usage` + `stop_reason` + `thinking` 是**同一个响应的不同部分**。

### 1.2 cCli 当前怎么处理的

把一个完整的 LLM 响应**拆成了互不关联的独立事件**：

```
LLM API 一次响应
  ├── thinking 内容  → 丢弃（完全没处理）
  ├── text 内容      → AgentEvent(text) → 累积到 accumulated → 最终写 assistant 事件
  ├── tool_use 调用  → AgentEvent(tool_start/tool_done) → 独立事件
  ├── usage tokens   → AgentEvent(llm_done) → 独立事件（llm_call_end in JSONL）
  ├── stop_reason    → AgentEvent(llm_done) → 独立事件
  └── refusal        → 丢弃（没处理）
```

结果：**assistant 事件只有纯文本，丢失了 usage、stop_reason、thinking、工具调用统计。**

### 1.3 日志中的实际表现

```
llm_call_end   { inputTokens: 11923, outputTokens: 91, stopReason: 'tool_use' }
                  ← 第一次 LLM 调用，要求调工具
tool_call_start { toolName: 'read_file' }
tool_call_end   { success: true }
llm_call_end   { inputTokens: 15668, outputTokens: 292, stopReason: 'end_turn' }
                  ← 第二次 LLM 调用，自然结束
assistant      { content: "你好...", model: "MiniMax-M2.5" }
                  ← 最终回复：不知道用了多少 token，不知道调了几次工具
```

## 二、改造目标

### 2.1 assistant 事件增强

```json
{
  "type": "assistant",
  "message": {
    "role": "assistant",
    "content": "你好，我来帮你...",
    "model": "MiniMax-M2.5",
    "provider": "minimax",
    "usage": {
      "inputTokens": 27591,
      "outputTokens": 383,
      "cacheReadTokens": 0,
      "cacheWriteTokens": 0
    },
    "stopReason": "end_turn",
    "llmCallCount": 2,
    "toolCallCount": 1
  }
}
```

### 2.2 thinking 内容支持

CLI 和 Web 都要能展示 LLM 的思考过程（Claude extended thinking / OpenAI reasoning）。

**CLI 端**：

```
◆ cCli (anthropic/claude-sonnet-4-6)
  💭 思考中...
  ├ 让我分析一下这个问题，用户要的是...
  ├ 考虑方案 A：直接修改文件
  └ 方案 B 更好：先读取再修改

  根据分析，我建议采用方案 B...
```

**Web 端**：折叠式展示，默认收起，点击展开。

```
[anthropic] claude-sonnet-4-6
┌─ 💭 思考过程 (点击展开)                    ▶
└─────────────────────────────────────────

根据分析，我建议采用方案 B...

27.6K in / 383 out · 2 次调用 · 1 次工具
```

## 三、改动设计

### 3.1 StreamChunk 加 thinking 类型

```diff
 export interface StreamChunk {
-  type: 'text' | 'tool_call' | 'usage' | 'done' | 'error'
+  type: 'text' | 'tool_call' | 'usage' | 'done' | 'error' | 'thinking'
   text?: string
+  thinking?: string
   toolCall?: ToolCallContent
   usage?: TokenUsage
   error?: string
   stopReason?: string
 }
```

### 3.2 AgentEvent 加 thinking 事件

```diff
 export type AgentEvent =
   | { type: 'text'; text: string }
+  | { type: 'thinking'; text: string }
   | { type: 'tool_start'; ... }
```

### 3.3 Anthropic Provider 透传 thinking

```typescript
// anthropic.ts 流式事件处理
if (event.type === 'content_block_delta') {
  if (event.delta.type === 'text_delta') {
    yield { type: 'text', text: event.delta.text }
  }
  // 新增：thinking 内容透传
  if (event.delta.type === 'thinking_delta') {
    yield { type: 'thinking', thinking: event.delta.thinking }
  }
}
```

注意：extended thinking 需要在请求中加 `thinking` 配置。当模型不支持或未启用时不会有 thinking 事件，完全兼容。

### 3.4 useChat 累积统计

```typescript
let accumulated = ''
let thinkingAccumulated = ''
let turnInputTokens = 0
let turnOutputTokens = 0
let turnCacheReadTokens = 0
let turnCacheWriteTokens = 0
let turnToolCallCount = 0
let turnLlmCallCount = 0
let lastStopReason = ''

for await (const event of loop.run(history)) {
  if (event.type === 'text') {
    accumulated += event.text
  }
  if (event.type === 'thinking') {
    thinkingAccumulated += event.text
  }
  if (event.type === 'llm_done') {
    turnInputTokens += event.inputTokens
    turnOutputTokens += event.outputTokens
    turnCacheReadTokens += event.cacheReadTokens
    turnCacheWriteTokens += event.cacheWriteTokens
    turnLlmCallCount++
    lastStopReason = event.stopReason
  }
  if (event.type === 'tool_done') {
    turnToolCallCount++
  }
  // ...
}

// 写入增强的 assistant 事件
sessionLogger.logAssistantMessage(accumulated, currentModel, currentProvider, {
  usage: { inputTokens: turnInputTokens, outputTokens: turnOutputTokens,
           cacheReadTokens: turnCacheReadTokens, cacheWriteTokens: turnCacheWriteTokens },
  stopReason: lastStopReason,
  llmCallCount: turnLlmCallCount,
  toolCallCount: turnToolCallCount,
  thinking: thinkingAccumulated || undefined,
})
```

### 3.5 logAssistantMessage 增强

```typescript
interface AssistantMeta {
  usage?: TokenUsage
  stopReason?: string
  llmCallCount?: number
  toolCallCount?: number
  thinking?: string
}

logAssistantMessage(content: string, model?: string, provider?: string, meta?: AssistantMeta): void {
  const msg: Record<string, unknown> = { role: 'assistant', content }
  if (model) msg['model'] = model
  if (provider) msg['provider'] = provider
  if (meta?.usage) msg['usage'] = meta.usage
  if (meta?.stopReason) msg['stopReason'] = meta.stopReason
  if (meta?.llmCallCount) msg['llmCallCount'] = meta.llmCallCount
  if (meta?.toolCallCount) msg['toolCallCount'] = meta.toolCallCount
  if (meta?.thinking) msg['thinking'] = meta.thinking
  this.#appendEvent('assistant', { message: msg })
}
```

## 四、LLM 内容类型全景

| 内容类型 | Anthropic | OpenAI | 当前处理 | 改造后 |
|---------|-----------|--------|---------|--------|
| **文本** | `content[].type = "text"` | `message.content` | ✅ 累积 | ✅ 不变 |
| **工具调用** | `content[].type = "tool_use"` | `message.tool_calls[]` | ✅ tool_start/done | ✅ 不变 |
| **思考过程** | `content[].type = "thinking"` | Responses API reasoning | ❌ 丢弃 | ✅ thinking 事件 + CLI/Web 渲染 |
| **拒绝** | 无 | `message.refusal` | ❌ 丢弃 | ✅ 转为 error 事件 |
| **Token 统计** | `usage` | `usage` | ✅ llm_done 事件 | ✅ 额外写入 assistant |
| **结束原因** | `stop_reason` | `finish_reason` | ✅ llm_done 事件 | ✅ 额外写入 assistant |
| **引用** | `citations` | 无 | ❌ 未处理 | 🔲 后续（P3） |
