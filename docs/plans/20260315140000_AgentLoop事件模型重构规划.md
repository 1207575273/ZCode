# AgentLoop 事件模型重构规划

> 日期: 2026-03-15
> 状态: 待实施
> 优先级: P1
> 触发: Web UI 开发中发现事件语义模糊，缺少 LLM 调用结束事件

---

## 一、问题描述

### 1.1 当前事件模型

```typescript
export type AgentEvent =
  // 业务事件（UI 消费）
  | { type: 'text'; text: string }
  | { type: 'tool_start'; toolName: string; toolCallId: string; args: Record<string, unknown> }
  | { type: 'tool_done'; toolName: string; toolCallId: string; durationMs: number; success: boolean; resultSummary?: string; meta?: ToolResultMeta }
  | { type: 'permission_request'; toolName: string; args: Record<string, unknown>; resolve: (allow: boolean) => void }
  | { type: 'user_question_request'; questions: UserQuestion[]; resolve: (result: UserQuestionResult) => void }
  | { type: 'error'; error: string }
  | { type: 'done' }
  // 观测事件（SessionLogger 消费）
  | { type: 'llm_start'; provider: string; model: string; messageCount: number; systemPrompt?: string }
  | { type: 'llm_usage'; inputTokens: number; outputTokens: number; cacheReadTokens: number; cacheWriteTokens: number; stopReason: string }
  | { type: 'llm_error'; error: string; partialOutputTokens?: number }
  | { type: 'tool_fallback'; toolName: string; fromLevel: string; toLevel: string; reason: string }
  | { type: 'permission_grant'; toolName: string; always: boolean }
  // 子 Agent
  | { type: 'subagent_progress'; agentId: string; description: string; turn: number; maxTurns: number; currentTool?: string }
```

### 1.2 问题清单

| 问题 | 说明 | 影响 |
|------|------|------|
| **缺少 `llm_done`/`llm_end`** | `llm_usage` 实际充当了"单次 LLM 调用结束"的角色，但名字只暗示"token 统计" | Web 端无法明确知道 LLM 调用何时结束，hook 系统无法在 LLM 结束后执行逻辑 |
| **`llm_usage` 语义模糊** | 既是"调用结束信号"又是"token 统计数据"，一个事件承担两个职责 | 不符合单一职责，消费者困惑 |
| **`done` 粒度太粗** | `done` 标志整个 AgentLoop 多轮执行结束，但不区分是正常结束还是因为 maxTurns 限制 | Web 端无法展示"达到最大轮次"的提示 |
| **缺少 `turn_start`/`turn_end`** | 多轮执行中无法区分每一轮的边界 | Web 端无法展示轮次信息，调试困难 |
| **事件分类不清晰** | "业务事件"和"观测事件"没有类型层面的区分，靠注释 | 消费者不知道哪些事件需要处理，哪些可以忽略 |

### 1.3 实际场景问题

**多轮 Agent 执行**的事件流：

```
llm_start → text* → llm_usage → tool_start → tool_done → llm_start → text* → llm_usage → done
              ↑                       ↑                       ↑                       ↑
         第一轮 LLM            第一轮工具              第二轮 LLM            整轮结束
         但没有明确的            有 start/done         同样没有明确的
         "LLM 结束"标志                                "LLM 结束"标志
```

对比 **工具事件**（有清晰的生命周期）：

```
tool_start → tool_done（或 tool_error 隐含在 tool_done.success=false）
```

**LLM 事件**缺少对称性：

```
llm_start → ???（缺 llm_done）→ llm_usage（语义不对）
```

## 二、重构方案

### 2.1 新增事件类型

```typescript
// 新增：LLM 调用结束（正常完成）
| { type: 'llm_done'; inputTokens: number; outputTokens: number; cacheReadTokens: number; cacheWriteTokens: number; stopReason: string }

// 新增：轮次边界（可选，P2）
| { type: 'turn_start'; turn: number; maxTurns: number }
| { type: 'turn_end'; turn: number }

// 修改：done 增加结束原因
| { type: 'done'; reason: 'complete' | 'max_turns' | 'aborted' }
```

### 2.2 事件生命周期对称

重构后，LLM 和工具的生命周期完全对称：

```
LLM:  llm_start → text* → llm_done（含 token 统计）
                         → llm_error（失败时替代 llm_done）

工具: tool_start → tool_done（含 durationMs, success, resultSummary）

整轮: done（含 reason）
```

### 2.3 `llm_usage` 的处理

两种方案：

**方案 A（推荐）：改名为 `llm_done`**

```diff
- | { type: 'llm_usage'; inputTokens: number; outputTokens: number; ... }
+ | { type: 'llm_done'; inputTokens: number; outputTokens: number; ... }
```

- 最小改动：只改 type 名
- `llm_done` 自然包含 token 统计（就像 `tool_done` 包含 `durationMs`）
- 向后兼容：SessionLogger 映射 `llm_done` → `llm_call_end`（已有）

**方案 B：拆分为两个事件**

```
llm_done → { type: 'llm_done'; stopReason: string }
llm_usage → { type: 'llm_usage'; inputTokens: number; ... }（纯统计，done 之后发）
```

- 更清晰：结束信号和统计数据分开
- 但增加了事件数量，消费者需要同时监听两个

**推荐方案 A**：`llm_done` 既是结束信号又带 token 统计，跟 `tool_done` 模式一致。

### 2.4 完整的新事件模型

```typescript
export type AgentEvent =
  // ── LLM 生命周期 ──
  | { type: 'llm_start'; provider: string; model: string; messageCount: number; systemPrompt?: string }
  | { type: 'llm_done'; inputTokens: number; outputTokens: number; cacheReadTokens: number; cacheWriteTokens: number; stopReason: string }
  | { type: 'llm_error'; error: string; partialOutputTokens?: number }

  // ── 流式输出 ──
  | { type: 'text'; text: string }

  // ── 工具生命周期 ──
  | { type: 'tool_start'; toolName: string; toolCallId: string; args: Record<string, unknown> }
  | { type: 'tool_done'; toolName: string; toolCallId: string; durationMs: number; success: boolean; resultSummary?: string; meta?: ToolResultMeta }
  | { type: 'tool_fallback'; toolName: string; fromLevel: string; toLevel: string; reason: string }

  // ── 交互事件 ──
  | { type: 'permission_request'; toolName: string; args: Record<string, unknown>; resolve: (allow: boolean) => void }
  | { type: 'permission_grant'; toolName: string; always: boolean }
  | { type: 'user_question_request'; questions: UserQuestion[]; resolve: (result: UserQuestionResult) => void }

  // ── 执行控制 ──
  | { type: 'done'; reason?: 'complete' | 'max_turns' | 'aborted' }
  | { type: 'error'; error: string }

  // ── 子 Agent ──
  | { type: 'subagent_progress'; agentId: string; description: string; turn: number; maxTurns: number; currentTool?: string }
```

## 五、远期考虑（P2）

### 5.1 轮次事件

```typescript
| { type: 'turn_start'; turn: number; maxTurns: number }
| { type: 'turn_end'; turn: number; toolCallCount: number }
```

用途：
- Web 端展示"第 N 轮 / 共 M 轮"
- 调试时定位问题在哪一轮
- 子 Agent 已有 `subagent_progress` 带 turn，主 Agent 也应该有

### 5.2 事件分类标签

```typescript
interface AgentEvent {
  type: string
  category: 'llm' | 'tool' | 'interaction' | 'control' | 'subagent'
}
```

消费者可以按 category 过滤：
- UI 只关心 `tool` + `interaction` + `control`
- SessionLogger 关心所有
- TokenMeter 只关心 `llm`
- Bridge 需要按 category 决定是否序列化（`interaction` 含 resolve 回调需要处理）
