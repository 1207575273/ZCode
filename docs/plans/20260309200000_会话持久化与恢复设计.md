# F8 会话持久化与恢复 — 设计文档

> 日期: 2026-03-09 | 对应需求: F8 对话中断恢复 (P1)

---

## 1. 目标

让用户关闭 cCli 后可恢复之前的对话，继续上下文交流。参考 Claude Code 的 resume 机制，采用 JSONL 事件流存储 + 按项目分目录组织。

## 2. 设计决策

### 2.1 存储格式：JSONL 事件流（非 JSON 快照）

**选择理由：**
- **Append-only 写入** — 每个事件追加一行，断电只丢最后一行，不会损坏整个文件
- **天然日志** — JSONL 本身就是结构化日志，F9 Agent 日志系统可直接复用
- **Token 计量数据源** — assistant 事件的 `usage` 字段，F10 可直接聚合
- **增量写入** — 无需每轮全量覆写，性能好

### 2.2 目录结构：按项目路径分目录

```
~/.ccode/sessions/
  └── <project-slug>/                          # cwd 路径转 slug
      ├── 20260309153022317_<uuidv7>.jsonl     # 会话事件流
      ├── 20260309160815042_<uuidv7>.jsonl
      └── <uuidv7>/                            # 可选：大文件外置目录
          └── tool-results/
              └── <toolCallId>.txt
```

**项目 slug 转换规则：** 将 cwd 路径中的 `/` `\` `:` 替换为 `-`，与 Claude Code 一致。

示例：`D:\a_dev_work\claude_cli_z01` → `D--a-dev-work-claude-cli-z01`

### 2.3 SessionId：UUIDv7

- UUIDv7 前 48 bit 是毫秒级 Unix 时间戳，天然有序可排序
- Node 20 不原生支持 v7，使用 `uuid` 包（项目已有依赖）
- 文件名格式：`YYYYMMDDHHMMSSMMM_<uuidv7>.jsonl`（人可读 + 自然排序 + UUID 唯一性）

### 2.4 恢复时只加载 user + assistant 文本消息

ChatView 当前只渲染 `user`/`assistant`/`system` 角色消息。恢复时从 JSONL 中过滤 `type=user` 和 `type=assistant`，提取 `message.content` 文本，重建 `ChatMessage[]`。工具调用历史保留在 JSONL 中供日志系统查阅，不加载到 UI。

## 3. 事件结构

```typescript
/** JSONL 中每行的结构 */
interface SessionEvent {
  sessionId: string
  type: SessionEventType
  timestamp: string              // ISO 8601
  uuid: string                   // 本条事件 ID (UUIDv4 即可)
  parentUuid: string | null      // 上一条事件 ID（链式结构）
  cwd: string                    // 运行时工作目录
  gitBranch?: string             // 当时的 git 分支
  message?: {
    role: string
    content: string | MessageContent[]
    model?: string               // assistant 消息附带模型名
    usage?: TokenUsage           // assistant 消息附带 token 用量
  }
  toolCallId?: string            // tool_call / tool_result 关联
  toolName?: string
  error?: string                 // error 事件的错误信息
  durationMs?: number            // turn_duration 事件的耗时
  provider?: string              // 当前 provider 名称
  model?: string                 // 当前 model 名称
}

type SessionEventType =
  | 'user'            // 用户输入
  | 'assistant'       // LLM 回复（含 usage）
  | 'system'          // 系统消息（指令输出、切换确认等）
  | 'tool_call'       // 工具调用开始
  | 'tool_result'     // 工具调用结果
  | 'session_start'   // 会话创建
  | 'session_resume'  // 会话恢复
  | 'turn_duration'   // 单轮耗时统计
  | 'summary'         // 上下文压缩摘要（未来扩展）
```

## 4. 核心模块

### 4.1 SessionStore (`persistence/session-store.ts`)

职责：JSONL 文件的读写、列出历史会话、清理过期会话。

```typescript
class SessionStore {
  /** 创建新会话，返回 sessionId */
  create(cwd: string, provider: string, model: string): string

  /** 追加事件到 JSONL */
  append(sessionId: string, event: SessionEvent): void

  /** 从 JSONL 回放，提取 user/assistant 消息重建 ChatMessage[] */
  loadMessages(sessionId: string): SessionSnapshot

  /** 列出指定项目（或所有项目）的最近会话 */
  list(options?: { projectSlug?: string; limit?: number }): SessionSummary[]

  /** 清理过期会话（默认 30 天） */
  cleanup(retentionDays?: number): void
}

interface SessionSnapshot {
  sessionId: string
  provider: string
  model: string
  cwd: string
  messages: ChatMessage[]
}

interface SessionSummary {
  sessionId: string
  projectSlug: string
  firstMessage: string          // 首条用户消息（截断，用于列表展示）
  updatedAt: string             // 最后事件时间
  gitBranch: string
  fileSize: number              // JSONL 文件大小
  filePath: string              // 文件绝对路径
}
```

### 4.2 useChat 改造

- 新增 `sessionId` state，启动时通过 `SessionStore.create()` 生成
- 每个关键事件（user 输入、assistant 回复、工具调用）后调用 `SessionStore.append()`
- 新增 `loadSession(sessionId: string)` 方法：从 JSONL 恢复消息和 provider/model

### 4.3 ResumeCommand (`commands/resume.ts`)

- `/resume` 指令，返回 `action.type = 'show_resume_panel'`
- App.tsx 收到 action 后展示 ResumePanel

### 4.4 ResumePanel (`ui/ResumePanel.tsx`)

独立全屏面板，与 ModelPicker / McpStatusView 相同的互斥模式。

**面板内容：**

```
┌ Resume session ────────────────────────────────────┐
│                                                     │
│❯ 帮我实现一个简单的计算器功能                         │
│  4 minutes ago · main · 24.8KB                      │
│                                                     │
│  给我给这个文件 生产一段python代码打印hello word       │
│  18 minutes ago · main · 11.8KB                     │
│                                                     │
│  修复登录页面的样式问题                               │
│  2 days ago · feat/login · 8.2KB                    │
│                                                     │
│  ↑↓ 选择 · Enter 恢复 · Ctrl+A 所有项目 · Esc 取消  │
└─────────────────────────────────────────────────────┘
```

**交互：**

| 按键 | 行为 |
|------|------|
| `↑↓` | 导航选择 |
| `Enter` | 恢复选中会话 |
| `Ctrl+A` | 切换 当前项目 / 所有项目 |
| 输入文字 | 模糊搜索（匹配首条消息） |
| `Esc` | 关闭面板 |

**每条会话显示：**
- 首条用户消息（截断 50 字符）
- 相对时间（`4 minutes ago` / `2 days ago`）
- git branch
- JSONL 文件大小

默认显示当前项目最近 **10 条**会话。

### 4.5 CLI 入口 (`bin/ccli.ts`)

新增 `--resume [sessionId]` 参数：
- `ccode --resume` — 启动后自动弹出 ResumePanel
- `ccode --resume <id>` — 直接恢复指定会话（支持前缀匹配）

### 4.6 WelcomeScreen 改造

右栏 "Recent sessions" 区域显示最近 3 条会话摘要（首条消息 + 相对时间），提示用户 `/resume` 查看更多。

## 5. 数据流

```
正常启动：
  App mount → useChat → SessionStore.create() → 生成 sessionId
  → 用户对话 → 每个事件 append 到 JSONL

恢复（/resume）：
  /resume 指令 → show_resume_panel → ResumePanel 展示
  → SessionStore.list() 获取会话列表
  → 用户选择 → useChat.loadSession(sessionId)
  → SessionStore.loadMessages() 回放 JSONL
  → 恢复 messages/provider/model → ChatView 渲染历史 → 继续对话

恢复（--resume）：
  CLI 解析参数 → App(showResumeOnStart=true)
  → 自动弹出 ResumePanel → 后续同上

自动保存时机：
  1. session_start — 会话创建时
  2. user — 用户发送消息时
  3. assistant — LLM 回复完成时（含 usage）
  4. tool_call — 工具调用开始时
  5. tool_result — 工具调用完成时
  6. session_resume — 恢复会话时
```

---

## 6. 会话清理

- 默认保留 30 天，可通过 `config.json` 的 `session.retentionDays` 配置
- 每次启动时异步检查并删除过期的 JSONL 文件及其 tool-results 目录
- 清理不阻塞启动流程

---

## 7. 不做的事

- **不存完整 LLM API 响应** — 只存文本内容 + usage，不存 stop_reason 等元数据
- **不做 /save 命名** — YAGNI，后续有需求再加
- **不做会话全文搜索** — 按时间列出 + 首条消息模糊搜索足够
- **不做 SubAgent JSONL 分离** — F13 实现时再加
- **不做大文件外置** — 当前工具结果较小，后续膨胀时再加 tool-results 目录
