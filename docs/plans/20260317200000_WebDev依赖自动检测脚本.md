# Web Dev 依赖自动检测脚本

> 日期: 2026-03-17
> 状态: 已完成

---

## 一、问题

`pnpm dev:web` 和 `pnpm build:web` 依赖 `web/node_modules` 存在。如果首次 clone 项目或 `web/node_modules` 被删除，直接运行会报错。用户需要手动 `cd web && pnpm install`，体验不好。

## 二、方案

新增 `scripts/ensure-web.cjs` 脚本（CommonJS 格式，因为项目 `"type": "module"` 导致 `.js` 被当 ESM）：

```javascript
// scripts/ensure-web.cjs
const { existsSync } = require('fs')
const { execSync } = require('child_process')
const { join } = require('path')

const nodeModules = join(__dirname, '..', 'web', 'node_modules')

if (!existsSync(nodeModules)) {
  console.log('web/node_modules not found, installing...')
  execSync('pnpm install', { cwd: join(__dirname, '..', 'web'), stdio: 'inherit' })
}
```

**逻辑**：检测 `web/node_modules` 是否存在，不存在就自动 `pnpm install`。已存在直接跳过（~1 秒）。

## 三、Scripts 改动

```json
{
  "dev:web": "node scripts/ensure-web.cjs && tsx bin/ccli.ts --web",
  "build:web": "node scripts/ensure-web.cjs && cd web && pnpm build",
  "build:all": "tsup && node scripts/ensure-web.cjs && cd web && pnpm build"
}
```

## 四、注意事项

- 文件必须用 `.cjs` 后缀（项目 `"type": "module"`，`.js` 会被当 ESM 处理，`require` 不可用）
- `--prefer-offline` 和 `--frozen-lockfile` 在 Vite 运行时会因 esbuild 文件锁定报 `EPERM`，所以只在 `node_modules` 不存在时才安装
- 新增包后需要手动 `cd web && pnpm install`（脚本只检测目录存在性，不检测 lockfile 变化）
