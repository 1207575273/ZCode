# TaskOutput 后台任务输出工具 — 实现总结

> 日期: 2026-03-18
> 状态: 已完成
> 对标: Claude Code `TaskOutput` 工具

---

## 一、背景

ZCli 的 `bash` 工具支持 `run_in_background` 后台运行命令，但启动后只返回 PID，**无法读取输出**。原因是后台进程使用 `stdio: 'ignore'` 完全断开了 IO 流。

Claude Code 通过 `TaskOutput` 工具补全了这个闭环——启动后台任务后可以随时读取其 stdout/stderr。

本次实现补齐了这一能力。

## 三、核心设计

### 3.1 数据流

```
bash run_in_background
  → execa(stdio: 'pipe')
    → stdout/stderr.on('data') → appendOutput(pid, chunk)
      → process-tracker 缓冲区（上限 1MB，超限丢弃头部保留尾部）
    → exit → markDone(pid, exitCode) → 30s 后 unregisterProcess(pid)

task_output { pid }
  → getProcess(pid) → getOutput(pid) → 返回拼接后的输出
  → block=true: 200ms 间隔轮询 proc.done，超时返回当前输出
```

### 3.2 TaskOutput 工具接口

```typescript
interface TaskOutputArgs {
  pid: number        // 后台进程 PID（必填）
  block?: boolean    // 是否阻塞等待进程结束（默认 false）
  timeout?: number   // 阻塞超时 ms（默认 10000，上限 300000）
}
```

- `block=false`：立即返回当前已捕获的输出（进程可能仍在运行）
- `block=true`：轮询等待进程结束或超时，再返回全部输出

### 3.3 process-tracker 增强

```typescript
interface TrackedProcess {
  pid: number
  command: string
  cwd: string
  startedAt: Date
  // --- 新增 ---
  outputChunks: string[]    // stdout + stderr 输出片段
  outputBytes: number       // 累计字节数（截断判断用）
  exitCode?: number | null  // 退出码
  done: boolean             // 是否已结束
}
```

**输出缓冲策略**：
- stdout + stderr 合并到同一个 `outputChunks` 数组
- 上限 1MB，超出时从头部丢弃最早的 chunk
- 进程退出后 30 秒自动清理，避免内存泄漏

### 3.4 bash.ts 关键改动

| 改前 | 改后 |
|------|------|
| `stdio: 'ignore'` | `stdio: 'pipe'` |
| 进程退出 → 立即 `unregisterProcess` | 进程退出 → `markDone` + 30s 延迟清理 |
| 无输出捕获 | stdout/stderr `on('data')` → `appendOutput` |
| — | stdout/stderr `.unref()` 不阻塞父进程退出 |

## 四、测试

### 4.1 单元测试（10 个用例）

```bash
npx vitest run tests/unit/tool-task-output.test.ts
```

| 用例 | 验证点 |
|------|--------|
| 应拒绝无效 pid | 负数 → error |
| 应拒绝不存在的 pid | 未注册 → error |
| 应返回运行中进程的已有输出 | 非阻塞读取正确拼接 |
| 应返回已结束进程的输出和退出码 | exitCode 正确显示 |
| block 模式应等待进程结束 | 50ms 后 markDone → 拿到完整输出 |
| block 模式超时应返回当前输出 | 300ms 超时 → 返回部分输出 |
| 无输出时应提示暂无输出 | 空缓冲区 → 友好提示 |
| appendOutput 应累积输出 | 多次追加正确拼接 |
| markDone 应记录退出码 | done=true, exitCode 正确 |
| 对不存在的 pid 操作应静默忽略 | 不抛异常 |

### 4.2 手动集成测试

```bash
npx tsx tests/manual/test-task-output.ts
```

测试流程：
1. `bash run_in_background` 启动 `echo hello && sleep 1 && echo world && sleep 1 && echo done`
2. 500ms 后非阻塞读取 → 只拿到 `hello`（后续输出尚未产生）
3. 阻塞读取 → 等进程结束后拿到完整的 `hello\nworld\ndone`
4. 结束后再读一次 → 输出仍在（30s 清理窗口内）

实际输出：

```
=== 1. 启动后台进程 ===
Background process started (pid: 20324). Use task_output tool to read output, or kill_shell to stop it.

=== 2. 立即读取（非阻塞）===
[进程仍在运行]
hello

=== 3. 阻塞等待进程结束 ===
[进程已结束 (exitCode: 0)]
hello
world
done

=== 4. 进程结束后再读一次 ===
[进程已结束 (exitCode: 0)]
hello
world
done

✓ 测试完成
```
