# 输入历史翻阅功能

> 日期: 2026-03-23
> 状态: 一期已完成

---

## 一、需求

用户在输入框按 ↑↓ 键可以翻阅之前发送过的消息，回填到当前输入框中。
行为对标 bash/zsh 的历史命令翻阅，以及 Claude Code CLI 的输入历史。

---

## 二、交互设计

### 触发时机（与多行编辑共存）

| 光标位置 | ↑ 行为 | ↓ 行为 |
|----------|--------|--------|
| 第一行 | 翻历史（上一条） | 正常下移光标 |
| 最后一行 | 正常上移光标 | 翻历史（下一条） |
| 中间行 | 正常上移光标 | 正常下移光标 |
| 输入框为空 | 翻历史 | 翻历史 |

判断方式：
- 第一行 = `value.lastIndexOf('\n', cursorIndex - 1) === -1`
- 最后一行 = `value.indexOf('\n', cursorIndex) === -1`

### 状态模型

```
history = ["msg1", "msg2", "msg3"]   ← 从 chat messages 提取（旧→新）
draftRef = "用户正在打的字"            ← 当前未发送的内容

historyIndex:
  -1  → 显示 draftRef（最新一页）
   0  → 显示 history[末尾]（最近一条）
   1  → 显示 history[末尾-1]
   ...
```

### 完整交互流程

```
1. 输入框: "hello"        index=-1 (draft="hello")
2. ↑ → 显示 "msg3"       index=0, draft 保存为 "hello"
3. ↑ → 显示 "msg2"       index=1
4. ↓ → 显示 "msg3"       index=0
5. ↓ → 显示 "hello"      index=-1 (恢复 draft)
```

### 编辑场景

```
1. ↑ → 显示 "msg3"       index=0
2. 用户改成 "msg3 改了"   → draft="msg3 改了", index 重置为 -1
3. ↑ → 显示 "msg3"       index=0 (原始历史不变)
4. ↓ → 显示 "msg3 改了"  index=-1
```

核心原则：
- **历史数组只读**，永远不被修改
- **draft 是活的**，用户在浏览历史时编辑内容 → 更新 draft 并回到最新页
- 提交后 draft 清空、index 重置

### 边界情况

| 场景 | 行为 |
|------|------|
| ↑ 已到最旧一条 | 不动，保持当前显示 |
| ↓ 已在最新页（index=-1） | 不动 |
| 无历史记录时按 ↑ | 不触发，正常上移光标 |
| 提交后 | historyIndex=-1, draft='' |

## 三、实现

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/ui/InputBar.tsx` | 新增 `history` prop、`historyIndex` + `draftRef` 状态、`navigateHistory()` / `onUserEdit()` 逻辑 |
| `src/ui/App.tsx` | `useMemo` 提取 `messages.filter(role==='user')` 传给 InputBar |

### 关键代码

```typescript
// InputBar.tsx — 历史导航
function navigateHistory(newIndex: number) {
  if (newIndex < -1 || newIndex >= history.length) return
  if (historyIndex === -1) draftRef.current = value  // 保存 draft
  setHistoryIndex(newIndex)
  const newValue = newIndex === -1
    ? draftRef.current
    : history[history.length - 1 - newIndex]!
  onChange(newValue)
  setCursorIndex(newValue.length)
}

// App.tsx — 提取历史
const userHistory = useMemo(() => {
  return messages.filter(m => m.role === 'user').map(m => m.content)
}, [messages])
```

## 四、后续扩展方向

当前一期实现仅覆盖**当前会话内**的历史。后续可扩展：

| 方向 | 说明 | 优先级 |
|------|------|--------|
| **跨会话历史** | 从 SessionStore（JSONL 持久化）加载上一次会话的用户消息，启动时预填充到 history 数组头部 | P1 |
| **历史持久化** | 独立 `~/.ccode/input_history` 文件（类似 `.bash_history`），跨会话 + 跨项目可用 | P1 |
| **历史条数上限** | 限制内存中保留最近 N 条（如 500），避免长期使用后内存膨胀 | P1 |
| **历史去重（可选）** | 连续重复的相同输入只保留一条。当前设计为不去重（用户确认） | P2 |
| **历史搜索** | Ctrl+R 反向搜索历史（类似 bash），输入关键词模糊匹配 | P2 |
| **按项目隔离** | 不同工作目录维护各自的历史，避免项目间污染 | P3 |
| **启动预加载** | `--resume` 恢复会话时自动加载该会话的用户消息作为历史 | P1 |

### 跨会话加载的设计思路

```
启动时:
1. 如果有 --resume，从对应 session JSONL 提取 role=user 消息
2. 否则，从 ~/.ccode/input_history 加载最近 N 条
3. 合并为初始 history 数组传给 InputBar
4. 当前会话新消息 append 到数组末尾

持久化时机:
- 每次用户提交消息后 append 到 input_history 文件
- 进程退出时不需要额外处理（已经逐条写入）
```
