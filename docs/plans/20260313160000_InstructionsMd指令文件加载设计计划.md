# Instructions.md 指令文件加载 — 设计计划

> **目标**：兼容 Claude Code 的 CLAUDE.md 生态，支持多层级指令文件发现与注入，每层优先 CCODE.md，fallback 到 CLAUDE.md。

**架构**：新增 `InstructionsLoader` 模块，启动时静态扫描 4 个层级，组合为 `<instructions>` 标签段落注入 system prompt。

**技术栈**：Node.js fs（同步读取） + git CLI（获取 repo root）

---

## 阶段划分

### 一期（本次实现）：静态加载

启动时一次性扫描所有层级，加载指令文件，会话期间不再变更。

### 二期（后续）：子目录懒加载

> **注意**：此能力本期 **不实现**，仅记录设计意图。
>
> **动机**：monorepo 场景下，不同子目录可能有各自的 CLAUDE.md（如 `frontend/CLAUDE.md` 和 `backend/CLAUDE.md`），
> 一次性全量加载会浪费 token 且可能指令冲突。懒加载在工具首次访问子目录时才加载该目录的指令文件。
>
> **实现思路**：ToolContext 新增 `onDirectoryAccess` 回调 → 工具执行时通知 → InstructionsLoader 检查并追加 → 下一轮 LLM 调用携带新指令。
>
> **推迟原因**：
> 1. 大多数项目不需要子目录级指令
> 2. 实现复杂度高（侵入每个工具 + 动态更新 system prompt）
> 3. 优先保证核心流程稳定

---

## 文件发现策略

每个层级检查两个文件名，优先 CCODE.md，没有则 fallback 到 CLAUDE.md。**同层只取一个**。

| 层级 | 路径 | 说明 |
|------|------|------|
| 1. 全局用户级 | `~/.ccode/CCODE.md` → `~/.claude/CLAUDE.md` | 用户的全局编码习惯 |
| 2. 项目根 | `<git-root>/CCODE.md` → `<git-root>/CLAUDE.md` | 项目级指令（团队共享） |
| 3. 项目配置目录 | `<git-root>/.ccode/CCODE.md` → `<git-root>/.claude/CLAUDE.md` | 项目配置目录中的指令 |
| 4. 当前工作目录 | `<cwd>/CCODE.md` → `<cwd>/CLAUDE.md` | 仅当 cwd ≠ git-root 时 |

**去重**：如果 cwd === git-root，层级 4 跳过。路径去重，同一文件不会加载两次。

**git-root 获取**：`git rev-parse --show-toplevel`，失败则 fallback 到 cwd。

---

## 注入格式

```xml
<instructions source="~/.ccode/CCODE.md" level="global">
内容...
</instructions>

<instructions source="/project/CCODE.md" level="project">
内容...
</instructions>
```

## System Prompt 组合顺序

```
[Instructions 段落]     ← 新增
+
[Skills 段落]           ← 已有
= 最终 systemPrompt
```
