# F6 Skills 注入与卸载 — 实现计划

> 日期：2026-03-11 | 状态：待评审

---

## 1. 需求目标

为 cCli 实现完整的 Skills 系统，使 LLM 能根据场景按需加载领域知识和工作流指令。

**核心特性：**
- Skills 发现与解析（YAML frontmatter + Markdown）
- 渐进式加载（L1 元数据 → L2 完整正文 → L3 资源文件）
- `skill` 原子工具（LLM 自主调用加载 Skill 正文）
- `/skills` 斜杠命令（用户手动触发）
- System Prompt 注入（启动时注入 skill 列表）
- 内置 `commit` Skill（提交规范）

**不在本期范围：**
- Plugin 容器（Phase 2）
- Marketplace / 远程安装（Phase 3）
- Claude Code 格式兼容（Phase 3）
- Hooks 系统
- Agents 子目录

## 2. 架构设计

### 2.1 Skill 文件格式

```
~/.ccode/skills/commit/SKILL.md          # 用户级
./.ccode/skills/my-skill/SKILL.md        # 项目级
内置 skills（src/skills/commit/）        # 系统级
```

每个 Skill 一个目录，包含 `SKILL.md`：

```markdown
---
name: commit
description: Use when the user asks to commit changes or create a git commit
allowed-tools: Bash
user-invocable: true
---

# Commit Skill

[Markdown 正文 — LLM 的行为指令]
```

**Frontmatter 字段：**

| 字段 | 必需 | 说明 |
|------|------|------|
| name | ✅ | 小写+连字符，唯一标识 |
| description | ✅ | 触发条件描述，LLM 据此判断是否调用 |
| allowed-tools | 否 | 限制该 skill 可使用的工具 |
| user-invocable | 否 | 是否出现在 `/skills` 菜单，默认 true |

### 2.2 三层渐进式加载

```
┌─────────────────────────────────────────────────┐
│ L1: System Prompt 注入（启动时）                  │
│   "Available skills:                             │
│    - commit: Use when user asks to commit...     │
│    - debugging: Use when encountering bugs..."   │
│                                                  │
│   → LLM 知道有哪些 skills 可用                    │
├─────────────────────────────────────────────────┤
│ L2: Skill 工具调用（按需）                        │
│   LLM 调用 skill({ name: "commit" })             │
│   → 返回 SKILL.md 完整正文                        │
│   → LLM 按正文指令行事                            │
├─────────────────────────────────────────────────┤
│ L3: 资源文件引用（按需）                          │
│   Skill 正文引用 references/xxx.md               │
│   → LLM 用 read_file 读取                        │
└─────────────────────────────────────────────────┘
```

### 2.3 模块依赖关系

```
src/core/skills/
├── types.ts          # SkillMetadata, SkillStore 接口
├── parser.ts         # SKILL.md 解析（frontmatter + body）
├── store.ts          # 文件系统发现 + 多源分层（内置 > 用户 > 项目）
└── skill-tool.ts     # 注册为 ToolRegistry 的原子工具

src/skills/
└── commit/
    └── SKILL.md      # 内置 commit skill

src/core/
├── agent-loop.ts     # [修改] 注入 systemPrompt
├── bootstrap.ts      # [修改] 初始化 SkillStore

src/providers/
├── anthropic.ts      # [修改] 消费 systemPrompt
└── openai-compat.ts  # [修改] 消费 systemPrompt

src/commands/
└── skills.ts         # /skills 指令

src/ui/
├── App.tsx           # [修改] 注册 /skills 指令
└── useChat.ts        # [修改] 传递 systemPrompt 给 AgentLoop
```

### 2.4 优先级与覆盖规则

同名 Skill 的优先级：**项目级 > 用户级 > 内置**

```
内置 src/skills/commit/SKILL.md         → 基线
用户 ~/.ccode/skills/commit/SKILL.md     → 覆盖内置
项目 ./.ccode/skills/commit/SKILL.md     → 覆盖用户
```

## 3. System Prompt 注入设计

### 3.1 现有基础设施

| 组件 | 现状 | 需要改动 |
|------|------|---------|
| `ChatRequest.systemPrompt` | 字段已定义，未使用 | 无需改动 |
| `message-converter` | 已支持 system 角色 | 无需改动 |
| `anthropic.ts` provider | 未消费 systemPrompt | 需补充 |
| `openai-compat.ts` provider | 未消费 systemPrompt | 需补充 |
| `agent-loop.ts` #callLLM | 未传递 systemPrompt | 需补充 |
| `useChat.ts` submit | 过滤 system 消息 | 需传递 systemPrompt 给 AgentLoop |

### 3.2 注入链路

```
SkillStore.getSystemPromptSection()
  → 生成 skills 列表文本
  ↓
AgentLoop constructor(config: AgentConfig)
  → config.systemPrompt = skills列表 + 其他系统指令
  ↓
AgentLoop.#callLLM()
  → chatRequest.systemPrompt = this.#config.systemPrompt
  ↓
Provider.chat(request)
  → 将 systemPrompt 作为首条 system message 发送给 API
```

### 3.3 System Prompt 内容模板

```markdown
## Available Skills

The following skills are available. Use the `skill` tool to load a skill's full instructions when relevant.

- **commit**: Use when the user asks to commit changes or create a git commit
- **debugging**: Use when encountering bugs or unexpected behavior

To use a skill, call the `skill` tool with the skill name. The tool will return the skill's full instructions.
```

## 5. 实现细节

### 5.1 types.ts

```typescript
export interface SkillMetadata {
  /** 唯一标识，小写+连字符 */
  name: string
  /** 触发条件描述 */
  description: string
  /** SKILL.md 文件绝对路径 */
  filePath: string
  /** 来源：builtin / user / project */
  source: 'builtin' | 'user' | 'project'
  /** 限制可用工具 */
  allowedTools?: string[]
  /** 是否用户可通过 /skills 手动触发 */
  userInvocable?: boolean
}
```

### 5.2 parser.ts

```typescript
/**
 * 解析 SKILL.md：提取 YAML frontmatter + Markdown body
 * 不依赖外部 YAML 库，手写简易解析（参考 superpowers skills-core.js）
 */
export function parseSkillFile(content: string): {
  frontmatter: Record<string, unknown>
  body: string
}

export function toSkillMetadata(
  frontmatter: Record<string, unknown>,
  filePath: string,
  source: SkillMetadata['source'],
): SkillMetadata | null  // 校验必填字段，不合法返回 null
```

### 5.3 store.ts

```typescript
export class SkillStore {
  /** 发现所有 skills，按优先级合并 */
  async discover(): Promise<SkillMetadata[]>

  /** 获取指定 skill 的完整正文（去掉 frontmatter） */
  async getContent(name: string): Promise<string | null>

  /** 生成 system prompt 中的 skills 列表段落 */
  buildSystemPromptSection(): string
}
```

扫描路径：
1. 内置：`path.join(__dirname, '../../skills/*/SKILL.md')` （打包后相对路径）
2. 用户级：`~/.ccode/skills/*/SKILL.md`
3. 项目级：`${cwd}/.ccode/skills/*/SKILL.md`

### 5.4 skill-tool.ts

```typescript
export class SkillTool implements Tool {
  readonly name = 'skill'
  readonly description = 'Load a skill\'s full instructions by name. ' +
    'Use this when a task matches an available skill.'
  readonly parameters = {
    type: 'object',
    properties: {
      name: { type: 'string', description: 'The skill name to load' },
    },
    required: ['name'],
  }
  readonly dangerous = false

  async execute(args: Record<string, unknown>): Promise<ToolResult> {
    const content = await this.#store.getContent(args.name as string)
    if (!content) return { success: false, output: '', error: `Skill "${args.name}" not found` }
    return { success: true, output: content }
  }
}
```
