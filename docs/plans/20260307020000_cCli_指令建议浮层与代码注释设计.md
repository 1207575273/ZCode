## 2. CommandSuggestion — 方案 C 设计

### 2.1 交互流程

```
用户输入 "/"
  → App.tsx inputValue = "/" → suggestions = registry.getAll()
  → CommandSuggestion 浮层出现，第 0 项高亮

用户继续输入 "/cl"
  → suggestions 过滤为 [/clear]

用户按 ↑↓
  → suggestionIndex 更新（App.tsx useInput 拦截）

用户按 Enter
  → 执行高亮指令（等同于提交 "/clear"）
  → inputValue 清空，浮层关闭

用户按 Esc
  → inputValue 清空，浮层关闭
```

### 2.2 外观

```
  /clear    Clear current conversation
❯ /help     Show available commands       ← 高亮项（绿色 ❯ + 白色文字）
  /model    Switch AI model
```

- 左侧 `❯` 标记高亮行，其余行缩进两格
- command name 绿色，description dimColor
- 无边框（轻量，区别于 PermissionDialog/ModelPicker）
- 渲染位置：InputBar 正上方

### 2.3 过滤规则

| 输入 | 匹配逻辑 |
|------|---------|
| `/` | 显示全部注册指令 |
| `/cl` | name 或 aliases 中有以 `cl` 开头的 |
| `任意非/文本` | 不显示 |

匹配大小写不敏感（`.toLowerCase()` 比较）。

## 3. 架构变更

### 3.1 InputBar → 受控组件

```typescript
// 新增 props
interface InputBarProps {
  value: string           // 新增：受控值
  onChange: (v: string) => void  // 新增：变更回调
  onSubmit: (value: string) => void
  placeholder?: string
  disabled?: boolean
}
```

`App.tsx` 提升 `inputValue` state，传给 InputBar。

### 3.2 CommandSuggestion 组件（纯显示）

```typescript
interface SuggestionItem {
  name: string
  aliases?: readonly string[]
  description: string
}

interface CommandSuggestionProps {
  items: SuggestionItem[]    // 已过滤的建议列表
  selectedIndex: number      // 当前高亮行（App 管理）
}
```

**纯显示组件**：不含 `useInput`，不管理自身状态。导航逻辑全部在 App.tsx 的 `useInput` 中。

### 3.3 App.tsx 新增逻辑

**新增 state / ref：**

```typescript
const [inputValue, setInputValue] = useState('')
const [suggestionIndex, setSuggestionIndex] = useState(0)
const suggestionConsumedRef = useRef(false)  // Enter 防双触发
```

**suggestions 计算（useMemo）：**

```typescript
const suggestions = useMemo(() => {
  if (!inputValue.startsWith('/')) return []
  const query = inputValue.slice(1).toLowerCase()
  return registry.getAll().filter(cmd =>
    cmd.name.startsWith(query) ||
    cmd.aliases?.some(a => a.startsWith(query))
  )
}, [inputValue, registry])
```

**useInput（仅建议可见时激活）：**

```typescript
useInput((_input, key) => {
  if (key.upArrow)   setSuggestionIndex(i => Math.max(0, i - 1))
  if (key.downArrow) setSuggestionIndex(i => Math.min(suggestions.length - 1, i + 1))
  if (key.return) {
    const cmd = suggestions[suggestionIndex]
    if (cmd) {
      suggestionConsumedRef.current = true  // 标记：阻止 InputBar.onSubmit 重复处理
      handleSubmit('/' + cmd.name)
      setInputValue('')
    }
  }
  if (key.escape) setInputValue('')
}, { isActive: suggestions.length > 0 })
```

**handleSubmit 防重复：**

```typescript
const handleSubmit = useCallback((input: string) => {
  // 建议项已由 useInput 处理，跳过 InputBar.onSubmit 的重复触发
  if (suggestionConsumedRef.current) {
    suggestionConsumedRef.current = false
    return
  }
  // ... 原有逻辑
}, [...])
```

**底部渲染：**

```tsx
{suggestions.length > 0 && (
  <CommandSuggestion items={suggestions} selectedIndex={suggestionIndex} />
)}
{pendingPermission != null
  ? <PermissionDialog ... />
  : showModelPicker
    ? <ModelPicker ... />
    : <InputBar value={inputValue} onChange={setInputValue} onSubmit={handleSubmit} disabled={isStreaming} />
}
```

### 3.4 suggestionIndex 重置

每当 `inputValue` 变化（过滤结果变化），`suggestionIndex` 重置为 0：

```typescript
useEffect(() => {
  setSuggestionIndex(0)
}, [inputValue])
```

## 4. 注释规范

### 4.1 文件级注释（每个文件顶部）

```typescript
/**
 * InputBar — 底部输入组件
 *
 * 受控文本输入框，用于接收用户消息或斜杠指令。
 * 通过 onChange 将当前值同步给父组件（App.tsx），

 */
```

 * dispatch() 解析 "/" 前缀输入并路由到对应指令。

## 6. 不做的事（YAGNI）

- 不做 Tab 自动补全（Enter 即执行，无需 Tab）
- 不做模糊匹配（前缀匹配足够）
- 不做建议项分组（当前指令数量少）
- 不给注释写测试
