# cCli 对话能力设计

> 日期: 2026-03-06 | 状态: **已确认**
> 覆盖需求: F1 (对话能力) + F2 (多模型支持)

---

## 1. 目标与范围

本期实现**纯文本流式对话**，用户输入消息后可与 Claude / GLM / GPT 等模型实时对话，逐 chunk 渲染回复。

**本期不包括：** 工具调用、Agent Loop、会话持久化、/model 切换命令。

---

## 4. Provider 实现

核心理念：**LangChain 作为底层引擎**，各 Provider 只是对 LangChain 类的薄封装，对外统一暴露 `LLMProvider` 接口。

### AnthropicProvider

```typescript
import { ChatAnthropic } from '@langchain/anthropic'

// 内部：new ChatAnthropic({ apiKey, model })
// 流式：model.stream(langchainMessages) → AsyncIterable<AIMessageChunk>
// 映射：chunk.content (string) → StreamChunk { type: 'text', text }
```

### OpenAICompatProvider

```typescript
import { ChatOpenAI } from '@langchain/openai'

// GLM: new ChatOpenAI({ apiKey, model, configuration: { baseURL } })
// GPT: new ChatOpenAI({ apiKey, model })
// 流式：model.stream() — 与 Anthropic 接口一致
```

GLM 只需传入 `baseURL: "https://open.bigmodel.ai/api/paas/v4"`，其余逻辑完全复用 OpenAI 兼容实现。

### 消息格式转换

```
我们的 Message (role: user/assistant/system)
  ↓
LangChain HumanMessage / AIMessage / SystemMessage
  ↓
model.stream()
  ↓
AIMessageChunk → StreamChunk { type: 'text', text: string }
stream 结束 → StreamChunk { type: 'done' }
出错 → StreamChunk { type: 'error', error: string }
```

## 5. 流式状态模型（`useChat` hook）

### 状态

```typescript
interface ChatState {
  messages: ChatMessage[]          // 历史消息（静态，已完成）
  streamingContent: string | null  // 当前流入文字，null = 空闲
  isStreaming: boolean             // true 时禁用输入框
  error: string | null             // 错误信息
}
```

### 动作

```typescript
interface ChatActions {
  submit: (text: string) => void   // 提交用户消息，触发流式调用
  abort: () => void                // 中断当前流式（AbortController）
}
```

### 流式生命周期

```
submit(text)
  ① isStreaming = true
  ② messages.push({ role: 'user', content: text })
  ③ streamingContent = ''           ← ChatView 渲染空气泡 + Spinner
  ④ 每个 text chunk:
       streamingContent += chunk    ← 逐 chunk 追加，打字机效果
  ⑤ done:
       messages.push({ role: 'assistant', content: streamingContent })
       streamingContent = null
       isStreaming = false
  ⑥ error:
       error = errorMsg
       streamingContent = null
       isStreaming = false
```

## 6. UI 改动

### ChatView 流式气泡

```
streamingContent == null     → 不渲染额外气泡
streamingContent == ''       → 渲染 "◆ cCli" 标签 + Spinner (●)
streamingContent != ''       → 渲染 "◆ cCli" 标签 + 已收到的文字
```

### InputBar 禁用态

- `disabled=true` 时 TextInput 仍渲染，但 onSubmit 被拦截不触发
- 视觉上提示符 `❯` 变为 dimColor

## 7. 决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| Provider SDK | LangChain (`@langchain/anthropic` / `@langchain/openai`) | 项目核心理念，统一抽象层 |
| 工具调用 | 本期不做 | YAGNI，先跑通对话主链路 |
| API Key 存储 | `~/.ccode/config.json` | 支持多 provider、自定义 baseURL（代理商/GLM） |
| 流式显示 | 参照 Claude Code：空气泡 + Spinner → 逐 chunk 追加 | 最佳用户体验 |
| Agent Loop | 本期用 `useChat` hook，下期拆为独立模块 | 避免过度设计 |
