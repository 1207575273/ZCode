# F9 Agent 日志系统实现总结

## 概述

F9 Agent 日志系统为 cCli 的 AgentLoop 引入了**事件流观察者模式**，将 LLM 调用、工具执行、MCP 连接等运行时行为以结构化事件写入 JSONL，实现完整的对话链路追踪和性能分析。

**核心设计**：AgentLoop 产生 AgentEvent → SessionLogger 订阅并映射为 SessionEvent → SessionStore 持久化到 JSONL。

---

## 架构

```
AgentLoop                   SessionLogger               SessionStore
  │                             │                           │
  ├─ yield llm_start ──────────► consume() ──────────────► append() → JSONL
  ├─ yield llm_usage ──────────► consume() ──────────────► append()
  ├─ yield tool_start ─────────► consume() ──────────────► append()
  ├─ yield tool_done ──────────► consume() ──────────────► append()
  │                             │                           │
  │                             ├─ logUserMessage() ─────► append()
  │                             ├─ logAssistantMessage() ─► append()
  │                             ├─ logMcpConnect() ──────► append()
  │                             └─ finalize() ───────────► append(session_end)
```

**关键特征**：
- **无侵入**：AgentLoop 通过 yield 产生事件，不依赖 SessionLogger
- **配对完整**：每种 `_start` 事件都有对应的 `_end` 事件
- **parentUuid 链**：所有事件通过 uuid/parentUuid 形成单链，支持事件树回溯
- **静默容错**：持久化失败不阻断主流程

---

## 事件类型映射

| AgentEvent (源) | SessionEvent (目标) | 说明 |
|-----------------|--------------------|----|
| `llm_start` | `llm_call_start` | LLM 调用开始，含 provider/model/messageCount |
| `llm_usage` | `llm_call_end` | LLM 调用结束，含 tokens 统计 |
| `llm_error` | `llm_call_end` | LLM 错误，stopReason='error' |
| `tool_start` | `tool_call_start` | 工具调用开始，含 toolName/args |
| `tool_done` | `tool_call_end` | 工具调用结束，含 durationMs/success |
| `tool_fallback` | `tool_fallback` | 工具降级记录 |
| `permission_grant` | `permission_grant` | 权限授予记录 |
| `error` | `error` | 运行时错误 |
| — | `session_end` | finalize() 写入，含累计统计 |
| `text` / `done` | (不写日志) | 纯 UI 事件 |

## exactOptionalPropertyTypes 兼容方案

TypeScript 严格模式下，`undefined` 不能赋值给可选属性。统一使用条件展开：

```typescript
// ✗ 错误：partialOutputTokens: tokens || undefined
// ✓ 正确：条件展开
...(event.resultSummary !== undefined ? { resultSummary: event.resultSummary } : {})
```

makeLlmError() 也采用三元表达式返回不同对象形状：
```typescript
return partialTokens > 0
  ? { type: 'llm_error', error, partialOutputTokens: partialTokens }
  : { type: 'llm_error', error }
```

## 决策记录

1. **SessionLogger 独立于 AgentLoop**：不在 AgentLoop 内部写日志，而是让 useChat 作为胶水层订阅事件流。好处：AgentLoop 保持纯粹的执行引擎角色，日志系统可独立测试。

2. **MCP 事件走回调而非 AgentEvent**：MCP 连接发生在 AgentLoop.run() 之前，不经过事件流。McpManager 通过 `onConnect` 回调直接通知 SessionLogger。

3. **单文件组织 agent-loop.ts**：用户明确拒绝拆分为两个文件，使用 ═══ 分区注释在单文件内清晰组织类型定义、类实现和工具函数。

4. **构造器注入 SessionStore**：SessionLogger 接受可选 store 参数，测试时传入临时目录的 store，生产代码使用默认单例。
