  ├── /session/:id     → ChatPage（聊天，已有）
  ├── /overview        → OverviewPage（总览大盘）
  ├── /conversations   → ConversationsPage（对话详情）

  ┌──────┐

### 2.1 数据流

```
前端页面 ──(fetch)──→ Bridge Server REST API ──→ 后端数据源
                                                   ├── SQLite (TokenMeter: usage_logs, pricing_rules)
                                                   ├── ConfigManager (config.json)
                                                   └── SessionStore (JSONL files)
```

聊天页面走 WebSocket 实时同步，管理页面走 REST API 请求-响应。

## 三、REST API 设计

### 3.1 总览大盘

```
GET /api/overview
  → { today: AggregateStats[], month: AggregateStats[], recentSessions: SessionSummary[] }
```

后端直接调 `tokenMeter.getTodayStats()` + `getMonthStats()` + `sessionStore.list()`。

### 3.2 对话详情

```
GET /api/conversations?limit=20&offset=0
  → { sessions: SessionSummary[], total: number }

GET /api/conversations/:sessionId
  → SessionSnapshot（含 messages + toolEvents）
```

后端调 `sessionStore.list()` + `sessionStore.loadMessages()`。

### 3.3 设置管理

```
GET /api/settings
  → { config: CCodeConfig, configPath: string }

PUT /api/settings
  Body: { config: Partial<CCodeConfig> }
  → { success: true }
```

后端调 `configManager.load()` + `configManager.save()`。

### 3.4 计价规则

```
GET /api/pricing
  → { rules: PricingRule[] }

POST /api/pricing
  Body: PricingRule (不含 id)
  → { id: number }

PUT /api/pricing/:id
  Body: Partial<PricingRule>
  → { success: true }

DELETE /api/pricing/:id
  → { success: true }
```

后端直接操作 SQLite `pricing_rules` 表。

## 七、技术要点

### 7.1 API 路由注册

```typescript
// src/bridge/api.ts
import { Hono } from 'hono'

export function createApiRoutes(): Hono {
  const api = new Hono()
  api.get('/overview', ...)
  api.get('/conversations', ...)
  api.get('/conversations/:id', ...)
  api.get('/settings', ...)
  api.put('/settings', ...)
  api.get('/pricing', ...)
  api.post('/pricing', ...)
  api.put('/pricing/:id', ...)
  api.delete('/pricing/:id', ...)
  return api
}

// server.ts 中注册
app.route('/api', createApiRoutes())
```

### 7.2 前端路由

```tsx
// App.tsx
<BrowserRouter>
  <div className="flex h-screen">
    <Sidebar />
    <div className="flex-1">
      <Routes>
        <Route path="/session/:sessionId" element={<ChatPage />} />
        <Route path="/overview" element={<OverviewPage />} />
        <Route path="/conversations" element={<ConversationsPage />} />
        <Route path="/conversations/:id" element={<ConversationDetailPage />} />
        <Route path="/settings" element={<SettingsPage />} />
        <Route path="/logs" element={<LogsPage />} />
        <Route path="/" element={<Navigate to="/overview" />} />
      </Routes>
    </div>
  </div>
</BrowserRouter>
```

### 7.3 Vite SPA fallback

Bridge Server 的 Vite 反代已经有 SPA fallback（`app.get('*', serveStatic(index.html))`）。react-router-dom 的客户端路由在 dev 和生产模式下都能正常工作。

### 7.4 计价规则 CRUD

直接操作 SQLite，不经过 ORM：

```typescript
// 查询
db.prepare('SELECT * FROM pricing_rules ORDER BY provider, priority DESC').all()

// 新增
db.prepare('INSERT INTO pricing_rules (...) VALUES (...)').run(...)

// 更新
db.prepare('UPDATE pricing_rules SET ... WHERE id = ?').run(...)

// 删除
db.prepare('DELETE FROM pricing_rules WHERE id = ?').run(id)
```
