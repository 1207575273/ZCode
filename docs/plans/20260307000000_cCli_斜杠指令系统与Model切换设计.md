# cCli 斜杠指令系统与 /model 切换 — 设计文档

> 版本: v1.0 | 日期: 2026-03-07
> 需求: F7 (斜杠指令系统骨架) + F14 (/model 交互式切换)

---

## 1. 目标范围

本次迭代实现：
- **CommandRegistry**：指令注册、解析、分发框架
- **内置指令**：`/help`、`/clear`、`/model`
- **ModelPicker UI**：方向键导航的模型选择弹窗
- **system 消息气泡**：ChatView 新增 `role: 'system'` 样式

本次**不包含**：
- 价格数据（所有模型价格显示 `--`，F10 完成后填入）
- `/skill`、`/plugin`、`/mcp`、`/resume`、`/config` 等其他指令（后续迭代）
- 指令持久化（切换的模型 session 级有效，不写回 config.json）

## 2. 架构设计

### 2.1 指令解析位置

指令解析在 **`App.tsx` 的 `handleSubmit`** 中进行：

```
InputBar 提交文本
  → App.tsx handleSubmit 检测 "/" 前缀
  → CommandRegistry.dispatch(input) → CommandResult
  → handleCommandAction(action) 更新 UI 状态
  → 非 "/" 文本 → useChat.submit()
```

理由：
- 指令不需要经过 LLM
- UI 状态变更（展示 ModelPicker 等）集中在 App 协调层
- 与 PermissionDialog 替换 InputBar 的已有模式完全一致

### 2.2 文件结构

```
src/commands/
├── types.ts          # CommandResult、CommandAction、Command 接口
├── registry.ts       # CommandRegistry 类
├── help.ts           # /help 指令实现
├── clear.ts          # /clear 指令实现
└── model.ts          # /model 指令实现

src/ui/
└── ModelPicker.tsx   # 模型选择弹窗组件
```

**修改文件：**
- `src/ui/App.tsx` — 指令拦截 + ModelPicker 渲染
- `src/ui/useChat.ts` — 新增 clearMessages / appendSystemMessage / switchModel / currentModel
- `src/ui/ChatView.tsx` — system 消息气泡样式

## 3. 核心类型

```typescript
// src/commands/types.ts

export type CommandAction =
  | { type: 'clear_messages' }
  | { type: 'show_help'; content: string }
  | { type: 'show_model_picker' }
  | { type: 'switch_model'; provider: string; model: string }
  | { type: 'error'; message: string }

export interface CommandResult {
  handled: boolean
  action?: CommandAction
}

export interface Command {
  name: string
  aliases?: string[]
  description: string
  execute(args: string[]): CommandResult
}
```

---

## 4. CommandRegistry

```typescript
// src/commands/registry.ts
class CommandRegistry {
  register(cmd: Command): void
  dispatch(input: string): CommandResult
  // 解析：去掉 "/" → 取第一个词为 cmd name → 剩余为 args
  // 未找到指令 → { handled: true, action: { type: 'error', message: 'Unknown command: /xxx' } }
  getAll(): Command[]
}
```

**解析规则：**
- `"/model"` → name=`"model"`, args=`[]`
- `"/model glm-5"` → name=`"model"`, args=`["glm-5"]`
- `"/model info"` → name=`"model"`, args=`["info"]`
- 支持 aliases（如 `/m` 等同于 `/model`）

## 5. 内置指令

### 5.1 /clear

```
/clear → { type: 'clear_messages' }
App.tsx → useChat.clearMessages() → setMessages([])
```

### 5.2 /help

动态从 CommandRegistry.getAll() 生成帮助文本：

```
Available commands:
  /help          Show this help message
  /clear         Clear current conversation
  /model         Switch AI model interactively
               /model <name>   Switch directly
               /model info     Show current model info
```

渲染为 `role: 'system'` 消息，在 ChatView 中以 dimColor + 左侧竖线样式展示。

### 5.3 /model

**子命令解析：**

| 输入 | 行为 |
|------|------|
| `/model` | → `show_model_picker` |
| `/model <name>` | → `switch_model`（直接切换） |
| `/model info` | → 展示当前模型信息卡片（system 消息） |

## 6. ModelPicker UI

**渲染位置：** 替换 InputBar（与 PermissionDialog 相同模式）

**外观：**
```
  当前模型: glm-5 (glm)

  可用模型:
  ❯ 1. glm-5            glm          --  / --
    2. glm-4-flash       glm          --  / --
    3. claude-sonnet-4-6 anthropic    --  / --

  ↑↓ 选择   Enter 确认   Esc 取消
```

**交互：**
- 初始光标停在当前激活模型
- ↑↓ 移动光标
- Enter → onSelect(provider, model)
- Esc → onCancel()

**模型数据来源：** `configManager.load()` 遍历 `config.providers`，每个 provider 下的 `models` 数组展开为列表项。

## 7. useChat 新增接口

```typescript
// useChat 新增返回字段
currentModel: string
currentProvider: string
clearMessages(): void
appendSystemMessage(text: string): void
switchModel(provider: string, model: string): void
```

**currentModel / currentProvider** 初始值从 `config.defaultModel` / `config.defaultProvider` 读取，切换时更新（session 级，不持久化）。

**appendSystemMessage** 追加 `{ id, role: 'system', content }` 到 messages 数组。

---

## 8. ChatView system 消息样式

新增 `role: 'system'` 消息的渲染：

```tsx
// role: 'system' 消息
<Box paddingLeft={1} borderLeft={true} borderColor="gray">
  <Text dimColor>{msg.content}</Text>
</Box>
```

与 user（右对齐蓝色）/ assistant（左对齐白色）区分，呈现为灰色系统提示。

## 9. App.tsx 底部区域渲染逻辑

```tsx
{pendingPermission != null
  ? <PermissionDialog ... />
  : showModelPicker
    ? <ModelPicker
        currentProvider={currentProvider}
        currentModel={currentModel}
        onSelect={(provider, model) => {
          switchModel(provider, model)
          appendSystemMessage(`✓ 已切换到 ${model} (${provider})`)
          setShowModelPicker(false)
        }}
        onCancel={() => setShowModelPicker(false)}
      />
    : <InputBar onSubmit={handleSubmit} disabled={isStreaming} />
}
```

## 10. 测试策略

- **CommandRegistry**：单元测试，覆盖 dispatch 解析、未知指令返回 error、aliases
- **/help**：验证生成文本包含所有注册指令
- **/clear**：验证 action 类型正确
- **/model**：验证子命令解析（无参数、带名称、info）
- **UI 组件**：无单元测试，通过手动冒烟验证
