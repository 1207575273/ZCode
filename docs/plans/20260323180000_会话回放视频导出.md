# 会话回放视频导出

> 日期: 2026-03-23
> 状态: 未完成（搁置）

---

## 一、需求

在 Web Dashboard 的历史对话详情页，提供导出按钮，将会话回放过程导出为视频文件，
便于分享、归档、演示。

---

## 二、已尝试的方案及问题

### 方案 A：自播放 HTML

生成一个独立 .html 文件，内含对话数据 + 打字机动画 JS。

**优点**：零依赖、极小、跨平台
**问题**：
- 工具消息等中间内容渲染不完整，需要在纯 JS 里重新实现 MessageBubble 的全部渲染逻辑
- 实际效果和 Web UI 差距大，维护两套渲染逻辑成本高

### 方案 B：html2canvas + MediaRecorder 录屏

用 html2canvas 定时截图消息区域 → 绘制到 canvas → captureStream + MediaRecorder 编码 WebM。

**优点**：录制的就是真实 DOM 渲染效果
**问题**：
- html2canvas 截图质量不稳定，部分 CSS（渐变、阴影、自定义字体）渲染异常
- 帧率低（4-5fps），文本动画不流畅
- html2canvas 体积大（~200KB gzipped），影响页面加载
- 长对话录制耗时过长

## 三、待探索的方案

| 方案 | 原理 | 优势 | 挑战 |
|------|------|------|------|
| **Puppeteer 服务端录制** | 后端启动无头浏览器，打开回放页面，用 `page.screencast()` 录制 | 效果 100% 还原 Web UI、帧率可控 | 需要服务端 Puppeteer/Playwright 环境 |
| **Canvas 自绘渲染** | 在 canvas 上手动绘制对话（文本、代码块、气泡），录制 canvas stream | 帧率高、体积小、无外部依赖 | 需要重写全部渲染逻辑，和 DOM 版本不一致 |
| **Web Codecs API** | 用 VideoEncoder 逐帧编码，替代 MediaRecorder | 更高质量、可输出 MP4 | 浏览器兼容性（Chrome 94+） |
| **Element Capture API** | Chrome 实验性 API，可直接捕获指定 DOM 元素的视频流 | 效果完美、代码极简 | 仅 Chrome，需要 flag 开启，尚未标准化 |
| **OBS/录屏工具集成** | 提供"一键全屏回放"模式，用户自己用 OBS 录制 | 零开发成本 | 用户体验差，需要额外工具 |

### 推荐方向

**短期（效果优先）**：Puppeteer 服务端录制，API 端点 `POST /api/conversations/:id/export-video`，
后端打开回放页 → 自动播放 → screencast → 返回视频文件。

**长期（标准化后）**：等 Element Capture API 进入稳定版，前端一行代码搞定。
