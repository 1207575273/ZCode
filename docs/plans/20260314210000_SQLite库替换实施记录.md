# SQLite 库替换实施记录：better-sqlite3 → libsql

> 日期: 2026-03-14
> 状态: 已完成
> 关联文档: `20260314200000_SQLite库替换评估与RAG能力规划.md`

## 一、变更目标

将 `better-sqlite3` 替换为 `libsql`，解除 Bun.js 迁移的唯一硬阻塞项（原生 C++ addon ABI 不兼容）。

## 二、变更范围

### 依赖变更

```diff
# package.json
- "better-sqlite3": "^12.6.2"
- "@types/better-sqlite3": "^7.6.13"
+ "libsql": "^0.5.22"

# pnpm onlyBuiltDependencies
- "better-sqlite3"
+ "libsql"
```

### 源码变更（3 个文件，仅 import 语句）

| 文件 | 改动 |
|------|------|
| `src/persistence/db.ts` | `import Database from 'better-sqlite3'` → `'libsql'` |
| `src/observability/token-meter.ts` | `import type { Database, Statement } from 'better-sqlite3'` → `'libsql'` |
| `src/core/cleanup-service.ts` | `import type { Database } from 'better-sqlite3'` → `'libsql'` |

**零业务逻辑变更** — libsql 的 API 与 better-sqlite3 完全兼容（`prepare/run/get/all/transaction/pragma/exec/close`），无需改动任何调用代码。

### 测试变更（3 个文件）

| 文件 | 改动 |
|------|------|
| `tests/unit/db.test.ts` | import → `libsql`，文件数据库改为 `:memory:` |
| `tests/unit/token-meter.test.ts` | import → `libsql`，文件数据库改为 `:memory:` |
| `tests/unit/core/cleanup-service.test.ts` | import → `libsql`，文件数据库改为 `:memory:` |

**测试改为内存数据库的原因**：libsql 在 Windows 上使用 WAL 模式时，`close()` 后仍持有 `-wal`/`-shm` 文件句柄，导致 `rmSync` 抛出 `EBUSY` 错误。内存数据库完全规避此问题，且测试速度更快。

## 四、libsql 关键特性

| 特性 | 说明 |
|------|------|
| **多运行时兼容** | Node.js + Bun.js + Deno 全兼容 |
| **API 兼容** | better-sqlite3 的 drop-in replacement |
| **向量搜索** | 内置 `F32_BLOB` 向量列 + DiskANN 索引（为 RAG 做准备） |
| **FTS5** | 全文搜索支持（SQLite 原生） |
| **纯本地** | 嵌入式运行，不依赖 Turso 云服务 |
| **维护方** | Turso (YC W23)，libSQL 是 SQLite 的开源 fork |
