# KillShell 工具设计与实施

> 日期: 2026-03-15
> 状态: 已完成

---

## 一、背景

BashTool 的 `run_in_background` 模式启动后台进程后，PID 只返回在文本提示中，无持久化追踪。LLM 要终止进程只能手动 `bash("kill PID")`——这不安全（可以杀任意系统进程）且不优雅。

Claude Code CLI 有专门的 kill 工具来管理后台进程生命周期。ZCli 缺少这个能力。

---

## 二、设计方案

### 2.1 架构

```
BashTool.run_in_background
  → execa(detached) → PID
  → processTracker.register(pid, command, cwd)
  → 进程退出时 → processTracker.unregister(pid)

KillShellTool.execute(pid)
  → processTracker.kill(pid) → 终止 + 取消注册

KillShellTool.execute()   // 无参数
  → processTracker.list() → 列出所有追踪中的后台进程
```

### 2.2 安全约束

- **只能终止追踪列表内的进程**（由 bash `run_in_background` 启动的）
- 不能传入任意系统 PID
- `dangerous = false`（终止自己启动的进程是安全行为）
- Windows 使用 `taskkill /F /PID xxx /T`（含子进程树）
- Unix 使用 `SIGTERM`（graceful shutdown）

## 三、新增文件

| 文件 | 职责 |
|------|------|
| `src/tools/process-tracker.ts` | 模块级进程注册表（register/unregister/list/kill/killAll） |
| `src/tools/kill-shell.ts` | `kill_shell` 原子工具 |
| `tests/unit/tool-kill-shell.test.ts` | 9 个测试用例 |

### 3.1 process-tracker.ts 接口

```typescript
registerProcess(pid, command, cwd)    // BashTool 后台启动时调用
unregisterProcess(pid)                // 进程退出时自动调用
listProcesses(): TrackedProcess[]     // 查询所有追踪进程
isTracked(pid): boolean               // 检查 PID 是否在追踪列表
killProcess(pid): { success, message } // 终止单个进程
killAllProcesses(): { killed, failed } // 终止全部
```

### 3.2 kill_shell 工具参数

```json
{
  "pid": {
    "type": "number",
    "description": "PID of the background process to kill. Omit to list all tracked processes."
  }
}
```

- 传 `pid` → 终止指定进程
- 不传 → 列出所有后台进程（PID + 运行时间 + 命令）

## 五、测试用例

| 用例 | 验证内容 |
|------|---------|
| 基础属性 | name = `kill_shell`，dangerous = false，有 pid 参数 |
| 空列表 | 无后台进程时返回 "No tracked background processes" |
| 列出进程 | bash 启动后台进程后，kill_shell 无参数能列出 |
| 终止进程 | kill_shell(pid) 成功终止 + 从追踪列表移除 |
| 未追踪 PID | kill_shell(999999) 返回失败提示 |
| 已退出进程 | 进程已自行退出后调 kill_shell 不崩溃 |
| PID 注册 | bash run_in_background 后 listProcesses 有记录 |
| 输出提示 | bash 后台输出包含 "kill_shell" 提示 |

### 对话方式验证

```
1. "帮我后台运行一个 sleep 30"          → bash(run_in_background: true)
2. "看看有哪些后台进程"                  → kill_shell()
3. "把刚才那个后台进程杀掉"              → kill_shell(pid: xxx)
4. "再看看还有没有后台进程"              → kill_shell() → 空列表
```
