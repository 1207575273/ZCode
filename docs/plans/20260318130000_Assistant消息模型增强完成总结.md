## 三、数据模型变化

### 3.1 JSONL assistant 事件

**改造前**：

```json
{
  "type": "assistant",
  "message": {
    "role": "assistant",
    "content": "你好...",
    "model": "glm-5",
    "provider": "glm"
  }
}
```

**改造后**：

```json
{
  "type": "assistant",
  "message": {
    "role": "assistant",
    "content": "根据分析，我建议...",
    "model": "claude-sonnet-4-6",
    "provider": "anthropic",
    "thinking": "让我分析这个问题...",
    "usage": {
      "inputTokens": 27591,
      "outputTokens": 383,
      "cacheReadTokens": 8004,
      "cacheWriteTokens": 0
    },
    "stopReason": "end_turn",
    "llmCallCount": 2,
    "toolCallCount": 1
  }
}
```

新增 5 个字段：`thinking`、`usage`（四维 token 累计）、`stopReason`、`llmCallCount`、`toolCallCount`。

### 3.2 AgentEvent 新增

```typescript
| { type: 'thinking'; text: string }
```

这是一个新的业务事件，从 Anthropic 的 `thinking_delta` 流式事件透传。OpenAI 兼容模型不产出此事件。

### 3.3 StreamChunk 新增

```typescript
type: 'text' | 'tool_call' | 'usage' | 'done' | 'error' | 'thinking'
thinking?: string
```

## 四、渲染效果

### 4.1 CLI 终端

```
◆ ZCli (anthropic/claude-sonnet-4-6)
  │ 💭 让我分析一下这个问题，用户要的是...考虑方案 A 和 B...

  根据分析，我建议采用方案 B，先读取文件内容再修改...
```

thinking 内容用黄色左竖线折叠显示，截断 200 字。正常文本在下方。

### 4.2 Web 端

```
[anthropic] claude-sonnet-4-6

┌─ 💭 思考过程 (156 字)               ▶ (点击展开)
└──────────────────────────────────────

根据分析，我建议采用方案 B...

────────────────────────────────────────
27.6K in / 383 out · 2 次调用 · 1 次工具
```

- thinking：`<details>` HTML 标签原生折叠，默认收起
- token 统计：底栏灰色小字，只在有 usage 数据时显示
- 多次调用/工具次数：只在 > 1 时显示

## 五、数据流链路

### 5.1 thinking 链路

```
Anthropic SDK
  → stream: content_block_delta { delta: { type: 'thinking_delta', thinking: '...' } }
    → AnthropicProvider: yield { type: 'thinking', thinking: '...' }
      → AgentLoop: yield { type: 'thinking', text: '...' }
        → useChat: thinkingAccumulated += event.text
        → eventBus.emit → Bridge → Web
          → Web: (暂不实时显示 thinking 流，只在 done 后显示最终值)
        → logAssistantMessage(meta: { thinking: thinkingAccumulated })
          → JSONL: { message: { thinking: '...' } }
            → loadMessages 还原 → Web/CLI 渲染
```

### 5.2 token 累计链路

```
AgentLoop #callLLM (第 1 次)
  → llm_done { inputTokens: 11923, outputTokens: 91 }
    → useChat: turnInputTokens += 11923, turnOutputTokens += 91, turnLlmCallCount++

AgentLoop #callLLM (第 2 次，带工具结果)
  → llm_done { inputTokens: 15668, outputTokens: 292 }
    → useChat: turnInputTokens += 15668, turnOutputTokens += 292, turnLlmCallCount++

AgentLoop done
  → useChat: logAssistantMessage(meta: {
      usage: { inputTokens: 27591, outputTokens: 383, ... },
      llmCallCount: 2,
      toolCallCount: 1
    })
    → JSONL: assistant 事件带完整累计
```
