# ZCli Agent Loop 与工具系统设计

> 日期: 2026-03-06 | 状态: **已确认**
> 覆盖需求: F4 (内置工具集)

---

## 1. 目标与范围

本期为 ZCli 添加工具调用能力，让 AI 能主动读写文件、搜索代码库、执行 Shell 命令。同时将现有 `useChat` hook 演进为真正的 Agent Loop 架构。

**本期包括：**
- Agent Loop 模块（支持多轮工具调用循环）
- 6 个内置工具：`read_file` / `write_file` / `edit_file` / `glob` / `grep` / `bash`
- 权限确认弹窗（危险工具需用户确认）
- 工具执行 UI（spinner → 完成状态）

**本期不包括：** MCP、/model 命令、会话持久化

## 2. Agent Loop 架构

### 核心设计

`AgentLoop` 是**纯逻辑类，不依赖 React**，通过 `AsyncIterable<AgentEvent>` 向外输出事件流，UI 层订阅事件更新 state。

### 执行流程

```
用户消息
  │
  ▼
AgentLoop.run(messages, tools)
  │
  ├─① LLM 调用（携带工具 schema）→ 流式接收
  │
  ├─ text chunk      → yield { type: 'text', text }
  │
  ├─ tool_call 收到  → yield { type: 'tool_start', toolName, args }
  │        │
  │        ├─ 安全工具 (read_file / glob / grep)
  │        │    → 直接执行
  │        │
  │        └─ 危险工具 (write_file / edit_file / bash)
  │             → yield { type: 'permission_request', toolName, args, resolve }
  │             → 等待 resolve(true/false)
  │             → true  → 执行工具
  │             → false → 跳过，结果标记为 "用户拒绝"
  │
  ├─ 工具执行完      → yield { type: 'tool_done', toolName, durationMs, success }
  │
  ├─ 工具结果追加到消息历史 → 回到 ① 继续循环
  │
  └─ done（无工具调用）→ yield { type: 'done' }，退出循环
```

### AgentEvent 类型定义

```typescript
type AgentEvent =
  | { type: 'text';               text: string }
  | { type: 'tool_start';         toolName: string; args: Record<string, unknown> }
  | { type: 'tool_done';          toolName: string; durationMs: number; success: boolean }
  | { type: 'permission_request'; toolName: string; args: Record<string, unknown>; resolve: (allow: boolean) => void }
  | { type: 'error';              error: string }
  | { type: 'done' }
```

## 3. 工具系统

### 工具接口（复用 `src/tools/types.ts`）

```typescript
interface Tool {
  name: string
  description: string
  parameters: Record<string, unknown>  // JSON Schema，传给 LLM
  execute(args: Record<string, unknown>, ctx: ToolContext): Promise<ToolResult>
}
```

### 6 个内置工具

| 工具 | 危险？ | 关键参数 | 说明 |
|------|:------:|----------|------|
| `read_file` | 否 | `path` | 读文件全文，超长自动截断 |
| `write_file` | **是** | `path`, `content` | 覆盖写文件，自动创建目录 |
| `edit_file` | **是** | `path`, `old_str`, `new_str` | 精确字符串替换，old_str 不存在时报错 |
| `glob` | 否 | `pattern`, `cwd?` | 文件路径匹配，返回路径列表 |
| `grep` | 否 | `pattern`, `path?`, `recursive?` | 内容搜索，返回匹配行（含行号） |
| `bash` | **是** | `command`, `cwd?` | 执行 shell 命令，走 ShellExecutor + 降级链 |

**危险工具常量：**
```typescript
const DANGEROUS_TOOLS = new Set(['write_file', 'edit_file', 'bash'])
```

### 工具注册中心 `src/tools/registry.ts`

```typescript
class ToolRegistry {
  register(tool: Tool): void
  getAll(): Tool[]                                    // 返回全部工具（传给 LLM schema）
  execute(name: string, args, ctx): Promise<ToolResult>
  isDangerous(name: string): boolean
}
```

## 4. UI 设计

### 工具执行状态展示

```
◆ ZCli
帮我读 src/ui/App.tsx 然后解释一下

⚙ Reading src/ui/App.tsx...     ← tool_start：spinner + 工具动作描述
✓ read_file (23ms)              ← tool_done：绿色 ✓ + 耗时

◆ ZCli
这个文件是...（流式回复）
```

工具动作描述从工具名 + 关键参数动态生成（如 `read_file {path}` → `Reading {path}...`）。

### 权限确认弹窗（替换 InputBar 位置）

```
┌─────────────────────────────────────────────┐
│ Bash command                                 │
│                                              │
│   mkdir -p "D:/a_dev_work/test1"            │
│   创建 test1 文件夹                          │
│                                              │
│  Do you want to proceed?                     │
│  ❯ 1. Yes                                   │
│    2. Yes, and don't ask again               │
│    3. No                                     │
└─────────────────────────────────────────────┘
```

- ↑↓ 移动选项，Enter 确认
- **选项 1（Yes）**：本次允许执行
- **选项 2（Yes, and don't ask again）**：加入 session 白名单，后续同工具不再询问
- **选项 3（No）**：拒绝，将拒绝信息回传给 LLM（LLM 可换方案继续）

白名单 `allowedTools: Set<string>` 存在 `useChat` state，session 级别，退出后清空。

## 7. 决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| Agent Loop 位置 | `src/core/agent-loop.ts`（纯逻辑） | 可独立测试，不依赖 React |
| 事件通信 | `AsyncIterable<AgentEvent>` | 与 LLM StreamChunk 模式一致，自然组合 |
| 权限弹窗选项 | Yes / Yes always / No | 参照 Claude Code，平衡安全与体验 |
| 白名单粒度 | 工具名级别，session 内有效 | 简单可靠，退出自动清空 |
| Bash 实现 | 走现有 ShellExecutor（Git Bash 优先） | 复用已有跨平台逻辑 |
