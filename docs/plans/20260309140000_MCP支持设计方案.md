# MCP 支持设计方案

> 状态：已确认 | 日期：2026-03-09

## 核心目标

实现 MCP Client，连接外部 MCP Server，动态扩展工具集。配置格式与 Claude Code 完全一致，做到无脑兼容。

## 配置文件

`~/.zcli/mcp.json`:

```json
{
  "mcpServers": {
    "mysql": {
      "command": "C:\\ProgramData\\anaconda3\\python.exe",
      "args": ["D:\\mcp-servers\\mysql-mcp\\src\\main.py"],
      "env": {
        "MYSQL_HOST": "10.0.0.4",
        "MYSQL_PORT": "3306"
      }
    },
    "streamable-mcp-server": {
      "type": "streamable-http",
      "url": "http://127.0.0.1:12306/mcp"
    },
    "puppeteer": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-puppeteer"]
    }
  }
}
```

**传输类型推断：**
- 有 `command` 字段 → stdio（默认，无需写 type）
- `type: "sse"` + `url` → SSE 传输
- `type: "streamable-http"` + `url` → Streamable HTTP 传输
- `env` 可选，注入子进程环境变量

## 工具命名

`mcp__<serverName>__<toolName>`（双下划线分隔，与 Claude Code 一致）

## 生命周期

- ZCli 启动时连接所有配置的 Server
- 进程退出时 graceful shutdown 断开所有连接
- 单个 Server 连接失败不影响其余

## 架构

```
McpManager
├── 读取 mcp.json
├── connectAll(): 启动时连接所有 Server
│   ├── stdio: spawn 子进程 + StdioClientTransport
│   ├── sse: SSEClientTransport(url)
│   └── streamable-http: StreamableHTTPClientTransport(url)
├── getTools(): 返回 McpTool[] 供注册
└── disconnectAll(): 退出时断开

McpTool implements Tool
├── name: "mcp__serverName__toolName"
├── dangerous: true（统一需确认）
├── parameters: 从 MCP inputSchema 转换
└── execute(): Client.callTool()
```

## 新建文件

| 文件 | 职责 |
|------|------|
| `src/mcp/mcp-manager.ts` | 配置读取、连接/断开、工具收集 |
| `src/mcp/mcp-tool.ts` | Tool 接口适配器 |
| `src/config/mcp-config.ts` | mcp.json 类型定义与读取 |

## 修改文件

| 文件 | 变更 |
|------|------|
| `src/ui/useChat.ts` | buildRegistry() 中注册 MCP 工具 |
| `src/ui/PermissionDialog.tsx` | MCP 工具 preview 显示 |

## 错误处理

| 场景 | 处理 |
|------|------|
| mcp.json 不存在 | 静默跳过 |
| 单个 Server 连接失败 | stderr 提示，跳过该 Server |
| 工具调用时 Server 已断开 | 返回 error result |
| mcp.json 格式错误 | 启动时报错，跳过 MCP |
