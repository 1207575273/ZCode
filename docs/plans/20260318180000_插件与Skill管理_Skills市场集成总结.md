# 插件与 Skill 管理 — Skills 市场集成总结

> 日期: 2026-03-18
> 状态: 已完成
> 前置: 原有 PluginsTab（Claude Code 导入）

---

## 一、背景

cCli 的插件管理此前只有一种获取方式——从本机已安装的 Claude Code 插件复制导入。这意味着：

1. **依赖 Claude Code 先行安装**，cCli 无法独立获取 skill
2. **无法接入社区生态**，skills.sh 上 275+ 个开放 skill 无从发现和使用
3. **没有统一的 skill 浏览入口**，用户需要自己搜索、手动下载、放到正确目录

本次迭代的目标是让 cCli 具备**独立的 Skill 获取能力**，直接对接 skills.sh 开放生态。

---

## 二、skills.sh 生态调研

### 2.1 技术架构

[skills.sh](https://skills.sh/) 是 Vercel 主导的开放 Agent Skills 生态系统：

- **CLI 工具**：`npx skills add <source>` — 安装 / 管理 / 搜索 skill
- **文件格式**：SKILL.md（YAML frontmatter + Markdown body）— **与 cCli 完全兼容**
- **安装源**：GitHub shorthand（`owner/repo`）、完整 URL、本地目录
- **目标 Agent**：40+ 个支持，包括 Claude Code、Cursor、Cline、Copilot 等

### 2.2 安装机制

`npx skills add` 执行后会在 `cwd` 下创建各 agent 的 skills 目录：

```
cwd/
├── .claude/skills/<name>/SKILL.md    ← Claude Code 格式
├── .cursor/skills/<name>/SKILL.md    ← Cursor 格式
├── .agents/skills/<name>/SKILL.md    ← 通用格式
└── ...
```

**关键限制**：不支持 `--output-dir` 参数，无法直接指定安装目录。

### 2.3 兼容性

cCli 的 SkillStore 扫描路径：
- `~/.ccode/plugins/<name>/skills/<skill>/SKILL.md` — 用户级插件
- `<cwd>/.ccode/plugins/<name>/skills/<skill>/SKILL.md` — 项目级插件

`npx skills` 安装的 SKILL.md 格式与 cCli 完全一致，只需搬到正确路径即可。

## 三、实现方案

### 3.1 安装流程

```
用户操作（热门推荐一键安装 / 命令行输入框）
  │
  ├─ POST /api/plugins/install-skill { source, skill? }
  │
  ├─ 1. mkdtemp 创建临时目录
  │     → 隔离 npx skills 的副作用
  │
  ├─ 2. npx skills add <source> --skill <name> --yes --copy
  │     → 在临时目录执行，skill 安装到 tempDir/.claude/skills/
  │
  ├─ 3. 扫描 tempDir/.claude/skills/ 下新增的 SKILL.md
  │     → 验证安装成功
  │
  ├─ 4. 搬运到 ~/.ccode/plugins/<repoName>/skills/<skillName>/
  │     → renameSync（同盘零拷贝）/ cpSync（跨盘回退）
  │
  ├─ 5. 写入 plugin.json 元数据
  │     → { name, description, version, source, installedAt }
  │
  └─ 6. 清理临时目录 → 返回结果
```

**为什么用临时目录而不是直接在 plugins 目录执行？**

`npx skills` 会在 cwd 下创建 `.claude/`、`.cursor/` 等多个 agent 目录。如果直接在 `~/.ccode/plugins/` 执行，会污染插件目录结构。临时目录隔离了这些副作用，只取我们需要的 `.claude/skills/` 部分搬运过来。

## 五、安装目录结构

安装 `vercel-labs/agent-skills --skill vercel-react-best-practices` 后：

```
~/.ccode/plugins/
└── vercel-labs--agent-skills/           ← repoName（/ 替换为 --）
    ├── plugin.json                      ← 元数据
    └── skills/
        └── vercel-react-best-practices/
            └── SKILL.md                 ← skill 内容
```

cCli 启动时 SkillStore 自动扫描 `~/.ccode/plugins/*/skills/*/SKILL.md`，新安装的 skill 下次启动即可使用。

## 六、异常处理

| 场景 | 处理 |
|------|------|
| skill 名称不匹配 | 检测输出含 "No matching skills found" → 返回 400 + 可用 skill 列表 |
| 网络超时 | execa timeout 120s → 返回 500 |
| 临时目录跨盘 | renameSync 失败 → cpSync 回退 |
| SKILL.md 不存在 | 安装完扫描为空 → 返回 400 + npx 输出 |
| 重复安装 | 目标目录已存在 → rmSync 后覆盖 |
