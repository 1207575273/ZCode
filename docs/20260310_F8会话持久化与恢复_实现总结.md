> 完成日期: 2026-03-10 | 需求优先级: P1 | 16 commits | 19 files | +2138 lines

## 1. 实现目标

让用户关闭 ZCli 后可恢复之前的对话，继续上下文交流。支持 JSONL 事件流存储、按项目分目录组织、交互式恢复面板、对话分叉（time-travel）。

---

## 2. 架构设计

### 2.1 存储格式：JSONL 事件流

采用 append-only 的 JSONL 格式（非 JSON 快照），每个事件追加一行。

**优势：**
- 断电只丢最后一行，不损坏整个文件
- 天然结构化日志，后续 Agent 日志系统可复用
- 增量写入，无需每轮全量覆写

### 2.2 目录结构

```
~/.zcli/sessions/
  └── <project-slug>/                       # cwd 路径转 slug（/\: → -）
      └── 20260310153022317_<uuidv7>.jsonl   # 时间戳前缀 + UUIDv7
```

### 2.3 事件树结构（支持分叉）

每个事件包含 `uuid`（自身 ID）和 `parentUuid`（父事件 ID），形成事件树：

```
session_start → user₁ → assistant₁ → user₂ → assistant₂        (主干)
                                     ↗ session_resume → user₃ → assistant₃  (分叉)
```

加载时从叶节点沿 `parentUuid` 回溯到根，精确还原任一分支。

## 4. 核心功能

### 4.1 自动持久化

用户发消息和 LLM 回复时自动追加事件到 JSONL，无需手动保存。Session 在首次 submit 时惰性创建。持久化失败不阻断对话流（静默处理）。

### 4.2 两种恢复入口

| 入口 | 行为 |
|------|------|
| `zcli --resume` | 启动后自动弹出 ResumePanel |
| `zcli --resume <id>` | 直接恢复指定会话 |
| `/resume` | 对话中打开 ResumePanel |

### 4.3 ResumePanel 交互

- **搜索框**（TextInput）：带光标，实时模糊过滤
- **Current Session**：第一项，Enter 返回当前对话
- **会话列表**：首条消息 + 相对时间 + git 分支 + 文件大小
- **分支子视图**：多分支会话自动展开分支选择
- **Ctrl+A**：切换当前项目 / 所有项目
- **Esc**：有搜索文字先清空，无则关闭

### 4.4 对话分叉（Time-Travel）

- `/fork` → 显示当前对话消息列表
- 选择任一消息作为分叉点 → 截断到该处 → 后续消息自然形成新分支
- `SessionStore.listBranches()` 通过叶节点检测自动识别所有分支
- `loadMessages(sessionId, leafEventUuid?)` 沿事件树精确加载指定分支

### 4.5 退出提示

Ctrl+C 退出时打印 resume 命令，支持开发模式（`pnpm dev --`）和全局安装（`zcli`）两种格式。使用 `process.on('exit')` 兜底确保可靠输出。

## 5. 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 存储格式 | JSONL 事件流 | Append-only 抗断电、增量写入、天然日志 |
| SessionId | UUIDv7 | 时间有序可排序，全局唯一 |
| 文件名 | `YYYYMMDDHHMMSSMMM_<uuid>.jsonl` | 人可读 + 自然排序 + UUID 唯一性 |
| 分叉机制 | parentUuid 事件树 | 无需额外数据结构，JSONL 天然支持 |
| 恢复范围 | 仅 user + assistant 文本 | 工具调用历史保留在 JSONL 但不加载到 UI |
| 模块级 sessionId | 与 mcpManager 同模式 | 退出时需在 React 组件树外访问 |
