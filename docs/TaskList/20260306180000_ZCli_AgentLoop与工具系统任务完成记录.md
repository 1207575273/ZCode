### T6: AgentLoop 核心
**Commit:** `394ab55`

**新建 `src/core/agent-loop.ts`：**

```typescript
export type AgentEvent =
  | { type: 'text';               text: string }
  | { type: 'tool_start';         toolName: string; toolCallId: string; args: Record<string, unknown> }
  | { type: 'tool_done';          toolName: string; toolCallId: string; durationMs: number; success: boolean }
  | { type: 'permission_request'; toolName: string; args: Record<string, unknown>; resolve: (allow: boolean) => void }
  | { type: 'error';              error: string }
  | { type: 'done' }

class AgentLoop {
  async *run(messages: Message[]): AsyncIterable<AgentEvent>
}
```

**核心循环（最多 20 轮）：**
1. 调用 LLM（携带所有工具 schema）
2. 收集 text chunk → yield text；收集 tool_call chunk → 暂存
3. 无工具调用 → yield done，退出
4. 对每个工具调用：
   - yield tool_start
   - 危险工具：`new Promise(r => resolvePermission = r)` + yield permission_request，await 等待用户
   - 执行工具，将结果追加到 history（作为 user 消息）
   - yield tool_done
5. 回到步骤 1 继续循环

**测试：** `tests/unit/agent-loop.test.ts` — 3 cases

### T8: useChat 演进 + 接线
**Commit:** `2acfcfc`

**重写 `src/ui/useChat.ts`，新增三个 state：**
```typescript
toolEvents: ToolEvent[]                           // 工具执行状态列表
pendingPermission: PendingPermission | null       // 待确认的权限请求
```

**关键实现细节：**
- `allowedToolsRef`（useRef）同步最新白名单，避免 async 闭包捕获陈旧值
- `pendingToolIds: Map<toolCallId, eventId>` 按 `toolCallId` 精确匹配 tool_done 事件
- `resolvePermission(allow, always)` 回调：`always=true` 时同步更新 ref + state

## 架构决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| Agent Loop 位置 | `src/core/agent-loop.ts`（纯逻辑） | 可独立测试，不依赖 React |
| 事件通信 | `AsyncIterable<AgentEvent>` | 与 LLM StreamChunk 模式一致，自然组合 |
| 权限弹窗暂停 | `new Promise(r => resolvePermission = r)` | 正确暂停 async generator，等待 UI resolve |
| 工具结果格式 | 追加为 `role: 'user'` 消息 | 简化实现，避免引入 LangChain ToolMessage 复杂依赖 |
| dangerous 标记位置 | 工具实例自带 `dangerous` 字段 | 内聚到工具定义本身，扩展性优于中央常量 Set |
| allowedTools 存储 | `useState` + `useRef` 双轨 | state 驱动 UI 重渲染；ref 让 async 闭包读取最新值 |
| tool_done 匹配 | 按 `toolCallId` | 同名工具多次调用时不会错误覆盖 |
| grep 正则 flag | `'i'`（无 `g`） | 带 `g` flag 的 `test()` 维护 lastIndex 状态，导致隔行跳过 |
| bindTools 格式 | OpenAI: function wrapper；Anthropic: input_schema | 两家 API schema 字段名不同，必须分别转换 |
