## 二、超出规划的增强

### 2.1 总览大盘（远超原规划）

**规划**：简单的 token 统计卡片 + 列表/表格。

**实际**：
- **四维 Token 卡片**：输入 / 输出 / 缓存读取 / 缓存写入 / 调用次数 / 费用（6 张卡片）
- **折线图（recharts）**：当日按小时 / 本周按天 / 本月按天 / 自定义按天
- **饼图（recharts）**：按供应商 Token 消耗分布
- **四个 Tab**：当日 / 本周 / 本月 / 自定义日期范围
- **自定义日期选择器**：开始日期 + 结束日期
- **模型用量明细表格**：8 列（供应商/模型/调用数/输入/输出/缓存读/缓存写/费用）
- **会话列表跟随 Tab**：按时间范围过滤

**规划只要了表格，实际做了完整的数据可视化大盘。**

### 2.2 对话详情

**规划**：会话列表 + 点击展开详情。

**实际额外增加**：
- **搜索框**：按 sessionId / 消息内容 / provider 实时模糊搜索
- **瀑布流回放**：进入详情自动逐条播放，消息淡入动画
- **回放控制栏**：播放/暂停/重播/显示全部 + 4 档速度（慢/中/快/瞬间）
- **进度显示**：`12 / 30 (40%)`
- **返回按钮改为 navigate(-1)**：从总览进详情返回总览，从列表进详情返回列表

### 2.3 设置管理

**规划**：Provider 列表 + 计价规则表格。

**实际额外增加**：
- **API Key 显隐切换**：默认脱敏，点击"显示"查看完整 key
- **计价规则新增表单**：9 个字段（供应商/模型匹配/四维价格/币种/生效日期/来源说明）
- **计价规则编辑**：点击"编辑"弹出表单，修改后保存

### 2.4 目录结构收敛

**规划**：api.ts 放在 `src/bridge/` 下。

**实际**：经过讨论，bridge + dashboard 收敛到 `src/server/` 统一入口：
```
规划: src/bridge/api.ts（与消息路由混放）
实际: src/server/
      ├── bridge/      ← WebSocket 消息路由
      └── dashboard/   ← REST API（关注点分离）
```

### 2.5 API 风格简化

**规划**：RESTful 风格（GET/POST/PUT/DELETE）。

**实际**：用户要求只用 GET + POST：
```
规划: PUT /api/settings       → 实际: POST /api/settings/save
规划: PUT /api/pricing/:id    → 实际: POST /api/pricing/update
规划: DELETE /api/pricing/:id → 实际: POST /api/pricing/delete
```

## 三、实际 REST API 清单（与规划对照）

| 规划 | 实际 | 差异 |

| `POST /api/pricing` | `POST /api/pricing/add` | 路径更明确 |
| `PUT /api/pricing/:id` | `POST /api/pricing/update` | 改 POST，body 含 id |
| `DELETE /api/pricing/:id` | `POST /api/pricing/delete` | 改 POST，body 含 id |

## 五、遇到的问题

| 问题 | 根因 | 修复 |
|------|------|------|
| Settings 页面白屏 | API Key 短于 8 字符，`repeat` 收到负数 | `Math.max(0, ...)` |
| 会话列表始终为空 | API 返回 `updatedAt`，前端用 `createdAt` 过滤 | 字段名对齐 |
| 图表/会话不跟 Tab | 数据源是全局的，没按 Tab 切换 | 每个 range 返回独立 trend + 前端按时间过滤 |
| 所有页面不能滚动 | App.tsx 主容器 `overflow-hidden` | 改为 `overflow-y-auto` |
| 自定义查询无变化 | 缺少调试手段 | useApi 加 console.log |
| 从总览进详情返回到列表 | `<Link to="/conversations">` 硬编码 | `navigate(-1)` |
