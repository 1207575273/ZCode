## 一、Dashboard 管理界面

### 1.1 页面总览

| 页面 | 路由 | 状态 |
|------|------|------|
| 聊天 | `/session/:id` | ✅ 已有，本期无改动 |
| 总览大盘 | `/overview` | ✅ 新增 |
| 对话历史 | `/conversations` | ✅ 新增 |
| 设置管理 | `/settings` | ✅ 新增（4 个 tab） |
| 日志浏览 | `/logs` | ✅ 占位 |

### 1.2 总览大盘

- **4 个时间范围 Tab**：当日 / 本周 / 本月 / 自定义（开始+结束日期选择器）
- **6 张统计卡片**：调用次数 / 输入 Token / 输出 Token / 缓存读取 / 缓存写入 / 费用
- **折线图（recharts）**：双 Y 轴——左轴 Token（蓝+绿实线），右轴费用（黄色虚线）
- **4 个饼图**：按供应商分布，分别展示输入 / 输出 / 缓存读 / 缓存写四个维度
- **模型用量明细表格**：8 列完整数据
- **会话列表**：跟随 Tab 时间范围过滤
- 当日折线图 X 轴按小时，本周/本月/自定义按天

### 1.3 对话历史

- **搜索框**：按 sessionId / 消息内容 / provider 实时模糊搜索
- **会话列表**：显示 sessionId + firstMessage 预览 + provider + 时间
- **瀑布流回放**：进入详情后消息逐条淡入显示
  - 4 档速度：慢(800ms) / 中(400ms) / 快(150ms) / 瞬间
  - 控制栏：播放 / 暂停 / 重播 / 显示全部
  - 进度显示：`12 / 30 (40%)`
- **返回按钮**：`navigate(-1)` 回退到来源页（从总览进来返回总览）

### 1.4 设置管理（4 个 Tab）

**Provider 配置**：
- defaultProvider 下拉选择（从已配置列表）+ 手动输入
- defaultModel 联动切换（选 provider 后自动列出其 models）+ 手动输入
- Provider 卡片列表：baseURL / API Key（显示/隐藏切换）/ 模型列表
- 保存按钮 → 写入 `~/.ccode/config.json` → Toast 通知 → **实时通知所有 CLI 实例刷新**

**计价规则**：
- 规则列表表格（供应商 / 模型匹配 / 输入价 / 输出价 / 币种 / 生效日期）
- 新增按钮 → 弹出 9 字段表单
- 编辑按钮 → 弹出表单预填数据
- 删除 → confirm 确认 → Toast 通知

**插件管理**：
- 已安装插件卡片列表（名称 / 版本 / Skills 数 / Hooks 有无 / 描述）
- 从 Claude Code 导入：检测 `~/.claude/plugins/installed_plugins.json` → 列出可导入的 → 逐个或全部导入
- 导入进度动画：⟳ 旋转 + "导入中..." → ✓ "已导入"
- 删除已安装插件

**MCP 管理**：
- MCP Server 列表（合并 `~/.ccode/.mcp.json` + `~/.claude.json` + `~/.mcp.json` 三层来源）
- 添加表单：4 种传输类型（Stdio / SSE / Streamable HTTP / HTTP）
  - Stdio：command + args
  - SSE / Streamable HTTP：url
  - HTTP：url + 自定义请求头(JSON)
- 只有 `~/.ccode/.mcp.json` 来源的 Server 可删除（其他来源只读）

### 1.5 通用 UI 组件

| 组件 | 用途 |
|------|------|
| Sidebar | 侧边导航（160px 文字模式，5 个页面入口） |

## 二、CLI 端改进

### 2.1 Assistant 消息标签

每条 AI 回复显示供应商和模型名：

```
◆ cCli (glm/glm-5)
你好！我有以下工具可用...
```

**实现**：
- `ChatMessage` 接口新增 `model` / `provider` 可选字段
- `useChat` 创建 assistant 消息时带上 `currentModel` / `currentProvider`
- `ChatView.MessageBubble` 在标签后显示 `(provider/model)`

**JSONL 落盘**：
- `logAssistantMessage(content, model, provider)` 三参数
- `message.provider` 新增字段，与 `message.model` 并列
- `loadMessages` 还原时提取两个字段

**设计意义**：为后续 LLM 池化 + 429 自动降级做准备。同一对话中可能有多个模型交替回复，每条消息标记清楚谁回的。

### 2.2 Web 端配置变更实时通知

```
Web 保存配置 → POST /api/settings/save
  → configManager.save() 写入文件
  → broadcastToClients(config_changed, 'cli')
  → Bridge Server 转发给所有 CLI
  → Bridge Client → EventBus(config_changed)
  → useChat 监听 → setCurrentProvider + setCurrentModel
  → CLI 显示 "配置已更新: glm / glm-5"
  → 下次 submit 用新 provider/model
```

**注意**：当前是全局配置，所有 CLI 实例都会收到。后续可做项目级配置隔离（`.ccode/settings.local.json` 覆盖全局）。

### 2.3 `/gc --days 0` 清理全部

原来 `--days 0` 被 `n > 0` 校验拒绝。改为 `n >= 0`，支持清理所有会话。

## 三、架构调整

### 3.1 目录收敛

```
原来:
  src/bridge/        ← 一级目录
  src/dashboard/     ← 一级目录

现在:
  src/server/
    ├── bridge/      ← WebSocket 消息路由
    │   ├── server.ts    (新增 broadcastToClients 导出)
    │   ├── client.ts    (新增 config_changed 处理)
    │   └── index.ts
    └── dashboard/   ← REST API
        ├── api.ts       (主路由 + overview/conversations/settings/pricing)
        ├── plugins-api.ts (插件管理)
        └── mcp-api.ts   (MCP 管理)
```

### 3.2 REST API 清单

| 方法 | 路径 | 用途 |
|------|------|------|
| GET | /api/overview | 总览（四维 token + 趋势 + 饼图 + 自定义范围） |
| GET | /api/conversations | 对话列表 |
| GET | /api/conversations/:id | 对话详情 |
| GET | /api/settings | 读取配置 |
| POST | /api/settings/save | 保存配置 + 广播 config_changed |
| GET | /api/pricing | 计价规则列表 |
| POST | /api/pricing/add | 新增规则 |
| POST | /api/pricing/update | 更新规则 |
| POST | /api/pricing/delete | 删除规则 |
| GET | /api/plugins | 已安装插件列表 |
| GET | /api/plugins/claude-available | Claude Code 可导入的插件 |
| POST | /api/plugins/import-claude | 从 Claude Code 导入 |
| POST | /api/plugins/delete | 删除插件 |
| GET | /api/mcp/servers | MCP Server 列表 |
| POST | /api/mcp/servers/add | 添加 MCP Server |
| POST | /api/mcp/servers/delete | 删除 MCP Server |

只使用 GET + POST，不用 PUT/DELETE。

### 3.3 新增依赖

| 包 | 用途 |
|----|------|
| react-router-dom | 多页面路由 |
| recharts | 折线图 + 饼图 |

## 四、遇到的问题

| 问题 | 根因 | 修复 |
|------|------|------|
| Settings 页白屏 | API Key < 8 字符，`repeat` 负数 | `Math.max(0, ...)` |
| 会话列表始终空 | API 返回 `updatedAt`，前端过滤用 `createdAt` | 字段对齐 |
| 图表/会话不跟 Tab | 数据源全局，没按 Tab 过滤 | 每个 range 独立数据 + 前端时间过滤 |
| 页面不能滚动 | `overflow-hidden` | 改 `overflow-y-auto` |
| 从总览进详情返回到列表 | `<Link>` 硬编码 | `navigate(-1)` |
| `/gc --days 0` 无效 | `n > 0` 拒绝了 0 | 改 `n >= 0` |
