# Ink 终端 Resize 响应式布局 — 经验总结

> 日期: 2026-03-06 | 项目: ZCli | 涉及文件: `src/ui/InputBar.tsx` / `src/ui/WelcomeScreen.tsx` / `src/ui/useTerminalSize.ts`

---

## 问题背景

拖动终端窗口调整大小时，出现两类问题：
1. InputBar 的 `─` 分隔线宽度不跟随终端变化（固定 60 字符）
2. WelcomeScreen 边框在 resize 结束后撕裂/错位

---

## 核心结论

**Ink 中有两类元素，响应式处理方式完全不同：**

| 元素类型 | 示例 | 正确做法 |
|----------|------|----------|
| Ink flexbox 容器（Box/边框） | WelcomeScreen 外边框 | 交给 Ink 布局引擎，用 `marginX` / `flexGrow` 即可 |
| 字符重复型内容 | `─`.repeat(n) 分隔线 | 必须用 state 驱动，监听 `stdout.resize` 事件 |

---

## 错误路径与教训

### 错误一：用 `stdout.columns` 直接读值（无响应性）

```tsx
// ❌ 错误：属性读取，不触发 re-render
const { stdout } = useStdout()
const separatorWidth = stdout?.columns ?? 80
```

`stdout.columns` 只是普通属性读取，不是响应式数据。即使 Ink 内部触发了 re-render，React 也不会因为这个值变化而重新渲染组件（没有 state 绑定）。

### 错误二：给 Box 加 `width={columns - 2}` 与 Ink 布局引擎竞争

```tsx
// ❌ 错误：手动管理 Box 宽度，与 Ink 内部 resize 机制竞争
const { columns } = useTerminalSize()
<Box width={columns - 2}>
```

**竞争时序（导致松开鼠标后画面撕裂）：**
```
拖动中：useTerminalSize setState → re-render → 宽度正确（两者同向）
松开后：Ink 内部 resize 重绘 + 我们的 state 停在上一个事件值 → 宽度不一致 → 撕裂
```

Ink 的 flexbox 布局引擎本身就会在 re-render 时用新的 `process.stdout.columns` 重新计算 Box 宽度，不需要外部干预。手动传入 `width` 反而引入了竞争。

---

## 正确方案

### 1. 需要字符精确计数的场景 → `useTerminalSize` hook + state

```ts
// src/ui/useTerminalSize.ts
export function useTerminalSize() {
  const { stdout } = useStdout()
  const [columns, setColumns] = useState(stdout?.columns ?? 80)
  const [rows, setRows] = useState(stdout?.rows ?? 24)

  useEffect(() => {
    if (!stdout) return
    const onResize = () => {
      setColumns(stdout.columns ?? 80)
      setRows(stdout.rows ?? 24)
    }
    stdout.on('resize', onResize)
    return () => { stdout.off('resize', onResize) }
  }, [stdout])

  return { columns, rows }
}
```

**原理：** `stdout.on('resize', ...)` 事件 → `setState` → React re-render → 新列数生效。

### 2. Ink Box 容器 → 只用 margin/flex，让 Ink 自己管

```tsx
// ✅ 正确：marginX 留边距，Ink flexbox 自动撑满终端宽度
<Box borderStyle="round" marginX={1} flexDirection="column">
```

当 InputBar 的 `columns` state 更新时，整个 App 树都会 re-render，WelcomeScreen 的 flexbox 在这次 re-render 中自然跟着更新。不需要 WelcomeScreen 自己持有宽度 state。

---

## 责任边界总结

```
useTerminalSize (state 驱动)
  └─ InputBar: '─'.repeat(columns)  ← 必须精确数字

Ink 布局引擎 (自动处理)
  └─ WelcomeScreen: marginX={1} + flexGrow={1}  ← 交给 Ink
  └─ App: width="100%"
```

---

## Windows 特殊说明

- Windows 没有 `SIGWINCH` 信号，但 Node.js 的 `process.stdout.on('resize', ...)` 在 Windows 下可用
- Ink 5 内部使用 `stdout.on('resize')` 而非 `SIGWINCH`，Windows 兼容
- 实测：拖动 Windows Terminal / CMD 窗口边缘时，resize 事件正常触发
