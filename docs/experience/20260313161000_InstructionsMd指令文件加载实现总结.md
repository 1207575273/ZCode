# Instructions.md 指令文件加载 — 实现总结

## 一、功能概述

新增 CCODE.md / CLAUDE.md 多层级指令文件发现与加载机制，兼容 Claude Code 的 CLAUDE.md 生态。每层优先查找 CCODE.md，没有则 fallback 到 CLAUDE.md，加载后注入 system prompt 供 LLM 使用。

**核心场景**：用户在全局或项目级配置编码规范、协作偏好等指令，LLM 在每次对话中自动遵循。

---

## 二、文件发现策略

每层只取一个文件（CCODE.md 优先），不同时加载同层的两个文件。

| 层级 | 路径 | 说明 |
|------|------|------|
| 1. 全局用户级 | `~/.ccode/CCODE.md` → `~/.claude/CLAUDE.md` | 用户的全局编码习惯 |
| 2. 项目根 | `<git-root>/CCODE.md` → `<git-root>/CLAUDE.md` | 项目级指令（团队共享） |
| 3. 项目配置目录 | `<git-root>/.ccode/CCODE.md` → `<git-root>/.claude/CLAUDE.md` | 项目配置目录 |
| 4. 当前工作目录 | `<cwd>/CCODE.md` → `<cwd>/CLAUDE.md` | 仅当 cwd ≠ git-root 时 |

**去重**：同一文件路径不会加载两次。

**git-root 获取**：`git rev-parse --show-toplevel`，失败则以 cwd 作为项目根。

---

## 三、注入格式与组合

### 3.1 标签包裹

```xml
<instructions source="C:\Users\xxx\.claude\CLAUDE.md" level="global">
文件内容...
</instructions>
```

### 3.2 System Prompt 组合顺序

```
[Instructions 段落]     ← 新增（指令文件内容）
+
[Skills 段落]           ← 已有（可用技能列表）
= 最终 systemPrompt
```

REPL 模式和 Pipe 模式均注入。

---

## 四、日志可观测

`llm_call_start` 事件新增 `systemPrompt` 字段，完整记录发送给 LLM 的 system prompt 内容到 JSONL 会话日志，便于排查和验证指令注入是否生效。

---

## 五、阶段规划

### 一期（本次实现）：静态加载

启动时一次性扫描所有层级，会话期间不变更，重启生效。

### 二期（后续）：子目录懒加载

monorepo 场景下，工具访问子目录时懒加载该目录的指令文件。本期不实现，原因：
1. 大多数项目不需要子目录级指令
2. 实现复杂度高（侵入每个工具 + 动态更新 system prompt）
3. 优先保证核心流程稳定
