## 一、产品定位

**cCli** 是一个开源的、支持多模型的 AI 编程 CLI 助手。对标 Claude Code 的交互体验，但不绑定单一 LLM 提供商。

**目标用户**：需要在终端环境中使用 AI 辅助编码、调试、重构的开发者。

**核心价值**：自由选择 Claude / GPT / Gemini / GLM / DeepSeek / 本地 Ollama 等任意模型，获得一致的 AI 辅助编码体验，同时拥有完全的可控性和可扩展性。

## 三、技术栈

### 3.1 CLI 核心

| 领域 | 技术 | 说明 |
|------|------|------|
| 运行时 | Node.js 20 + TypeScript 5 (strict) | 规划迁移 Bun.js |
| 终端 UI | Ink 5 + React 18 | CLI 组件化渲染 |
| LLM (Anthropic) | `@anthropic-ai/sdk` | 官方 SDK 直连 |

| MCP | `@modelcontextprotocol/sdk` | 官方 MCP 客户端 |

| 文件搜索 | `fast-glob` + `@vscode/ripgrep` | 高性能匹配和搜索 |
| 持久化 (对话) | JSONL 文件 | 可读、可调试、支持分支 |
| 持久化 (计费) | libsql (SQLite) | 多维聚合查询，兼容 Node/Bun/Deno |
| 测试 | Vitest | TS 原生支持 |
| 构建 | tsup | ESM 输出 |

### 3.2 Web 能力

| 领域 | 技术 | 说明 |

| 实时通信 | WebSocket (ws) | CLI ↔ Bridge ↔ Web 双向 |

| 组件 | 迁移改动 |

## 四、架构概览

### 4.1 系统分层

```
┌─────────────────────────────────────────────────┐
│              CLI Entry (bin/ccli.ts)             │
├─────────────────────────────────────────────────┤
│         Terminal UI Layer (Ink/React)            │
│  ChatView / InputBar / ToolStatusLine / ...     │
├─────────────────────────────────────────────────┤
│         Command & Slash System                   │
│  /help /model /clear /mcp /resume /fork /bridge │
├─────────────────────────────────────────────────┤
│         Agent Loop (Core Engine)                 │
│  AsyncGenerator<AgentEvent> 多轮执行引擎        │
│  LLM Call → Tool Execution → Next Turn          │
│  SubAgent Scheduler (dispatch_agent)            │
├──────────┬──────────┬───────────────────────────┤
│ Provider │  Tools   │  Skills & Plugins          │
│ Claude   │ BuiltIn  │  SKILL.md Prompt Skills   │
│ OpenAI   │ MCP      │  Plugin Packages          │
│ Gemini   │          │  (superpowers etc.)        │
│ Ollama   │          │                            │
├──────────┴──────────┴───────────────────────────┤
│              Bridge Layer                        │
│  EventBus (进程内) → BridgeClient (WS) →        │
│  Bridge Server (纯路由器) → Web (React SPA)     │
├─────────────────────────────────────────────────┤
│              Observability Layer                  │
│  SessionLogger (JSONL) / TokenMeter (SQLite)    │
├─────────────────────────────────────────────────┤
│              Persistence Layer                    │
│  SessionStore (JSONL) / ConfigManager / db.ts   │
└─────────────────────────────────────────────────┘
```

### 4.2 Bridge Server 架构（多 CLI 平等连接）

```
Bridge Server (port 9800) — 纯消息路由器
  │
  │  session registry: sessionId → [clients...]
  │
  ├── CLI-1 (ws client, session-111)
  ├── CLI-2 (ws client, session-222)
  ├── Web-A (ws client, session-111) → 只收 session-111 事件
  └── Web-B (ws client, session-222) → 只收 session-222 事件
```

### 4.3 Session 生命周期

```
CLI 启动 → useChat mount → ensureSession() → sessionId 立即可用
  → Bridge Client register → 路由生效
  → submit() → AgentLoop → 事件流 → SessionLogger / TokenMeter / EventBus
  → CLI 退出 → session_end 汇总 → 打印 resume 命令
  → ccode --resume <id> → 恢复完整历史
```

详见 `20260315040000_Session生命周期与项目架构.md`。
